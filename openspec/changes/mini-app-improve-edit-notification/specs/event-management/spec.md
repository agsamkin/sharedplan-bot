## MODIFIED Requirements

### Requirement: Уведомления участников при изменении/удалении

Все участники пространства (кроме владельца события) ДОЛЖНЫ получать детальное уведомление через Telegram бота при редактировании или удалении события из Mini App. Уведомление ДОЛЖНО содержать достаточно информации, чтобы участник понял суть изменения без необходимости открывать Mini App.

#### Scenario: Уведомление при редактировании одного поля

- **WHEN** владелец изменяет одно поле события через Mini App (например, дату)
- **THEN** всем участникам пространства (кроме владельца) отправляется сообщение, содержащее: название пространства, название события, изменённое поле со старым и новым значением (формат «старое → новое»), имя редактора

#### Scenario: Уведомление при редактировании нескольких полей одновременно

- **WHEN** владелец изменяет несколько полей события через Mini App (например, дату и время)
- **THEN** отправляется ОДНО сообщение со всеми изменёнными полями, каждое на отдельной строке в формате «старое → новое»

#### Scenario: Формат даты в уведомлении о редактировании

- **WHEN** владелец изменяет дату события
- **THEN** старая и новая дата отображаются в формате «день месяца, день недели» (например, «28 марта, суббота → 30 марта, понедельник»)

#### Scenario: Формат времени в уведомлении о редактировании

- **WHEN** владелец изменяет время события
- **THEN** старое и новое время отображаются в формате HH:MM (например, «19:00 → 20:30»)

#### Scenario: Уведомление при удалении события

- **WHEN** владелец удаляет событие через Mini App
- **THEN** всем участникам пространства (кроме владельца) отправляется сообщение, содержащее: название пространства, название удалённого события, дату и время удалённого события, имя удалившего

#### Scenario: Формат даты/времени в уведомлении об удалении

- **WHEN** владелец удаляет событие с датой и временем
- **THEN** уведомление содержит дату в формате «день месяца, день недели» и время в формате HH:MM на одной строке (например, «28 марта, суббота, 19:00»)

#### Scenario: Удаление события без времени

- **WHEN** владелец удаляет событие без указанного времени (на весь день)
- **THEN** уведомление содержит только дату без времени

#### Scenario: Ошибка отправки уведомления

- **WHEN** уведомление не удаётся отправить одному из участников
- **THEN** ошибка логируется, остальные уведомления продолжают отправляться
