## Why

Сейчас в Mini App экран EventDetailPage позволяет любому участнику пространства редактировать поля (название, дата, время) и удалять любое событие, хотя бэкенд уже возвращает ошибку 403 для не-владельцев. Пользователь видит форму редактирования и кнопку удаления, пытается сохранить — получает непонятную ошибку. Это создаёт путаницу и ложные ожидания. Для пространств эта проблема уже решена (SpaceDetailPage проверяет `isAdmin`), но для событий — нет.

## What Changes

- На экране EventDetailPage для не-владельца поля становятся **read-only** (информация отображается, но input-ы заблокированы)
- Кнопка **«Сохранить»** скрывается для не-владельца
- Кнопка **«Удалить событие»** скрывается для не-владельца
- Заголовок страницы меняется: для владельца — «Редактирование», для не-владельца — «Событие»
- Используется поле `is_owner` из ответа API `GET /api/events/:id`, которое уже возвращается бэкендом

## Impact

- **Frontend**: `mini_app/frontend/src/pages/EventDetailPage.tsx` — основные изменения: использование `is_owner` для условного рендеринга UI
- **Backend**: изменения не требуются — API уже возвращает `is_owner` в `GET /api/events/:id` и `GET /api/spaces/:id/events`, и уже проверяет владение в `PUT` и `DELETE`
- **API**: без изменений контракта
- **Модели**: без изменений
