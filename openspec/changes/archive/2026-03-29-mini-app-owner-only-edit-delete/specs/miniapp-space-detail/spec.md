## MODIFIED Requirements

### Requirement: Переработанный экран EventDetail

Система SHALL отображать экран просмотра/редактирования события по маршруту `/events/:id`. При загрузке события система SHALL использовать поле `is_owner` из ответа API для определения режима отображения.

**Режим владельца** (`is_owner === true`): экран SHALL содержать Section "Детали события" с редактируемыми полями Название (input), Дата (input type="date"), Время (input type="time"), Section с информацией об авторе (IconPerson + "Создал: имя"), кнопку "Сохранить" (primary blue), кнопку "Удалить событие" (danger outline). Удаление SHALL использовать инлайн-подтверждение. Заголовок Header SHALL быть "Редактирование".

**Режим просмотра** (`is_owner === false`): экран SHALL содержать Section "Детали события" с полями в режиме read-only (атрибут `readOnly` на input-элементах, приглушённый стиль без border), Section с информацией об авторе. Кнопки "Сохранить" и "Удалить событие" SHALL НЕ отображаться. Заголовок Header SHALL быть "Событие".

#### Scenario: Владелец открывает своё событие
- **WHEN** пользователь открывает экран `/events/:id` и API возвращает `is_owner: true`
- **THEN** отображается заголовок "Редактирование", поля редактируемые, кнопки "Сохранить" и "Удалить событие" видны

#### Scenario: Участник открывает чужое событие
- **WHEN** пользователь открывает экран `/events/:id` и API возвращает `is_owner: false`
- **THEN** отображается заголовок "Событие", поля в режиме read-only (readOnly, приглушённый стиль), кнопки "Сохранить" и "Удалить событие" отсутствуют

#### Scenario: Редактирование и сохранение владельцем
- **WHEN** владелец меняет название события и нажимает "Сохранить"
- **THEN** API `PUT /api/events/:id` обновляет событие, навигация назад, Toast "Событие обновлено"

#### Scenario: Удаление события владельцем
- **WHEN** владелец нажимает "Удалить событие" и подтверждает
- **THEN** API `DELETE /api/events/:id` удаляет событие, навигация назад, Toast "Событие удалено"

#### Scenario: Не-владелец не может редактировать через UI
- **WHEN** не-владелец находится на экране события с `is_owner: false`
- **THEN** input-поля имеют атрибут `readOnly`, кнопки управления отсутствуют в DOM, попытка взаимодействия с полями не приводит к изменению значений
