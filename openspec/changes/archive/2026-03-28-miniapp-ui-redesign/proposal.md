## Why

Текущий UI Mini App использует стандартные компоненты `@telegram-apps/telegram-ui`, что даёт базовый внешний вид, но не обеспечивает профессионального UX: нет визуальной иерархии событий (дата-бейджи, относительные даты), отсутствуют инлайн-подтверждения удаления, нет тостов обратной связи, нет аватаров с инициалами, а экран пространства совмещает настройки и события без чёткого разделения. Редизайн по готовым макетам повысит удобство и визуальное качество приложения.

## What Changes

- **Замена компонентной базы**: вместо `@telegram-apps/telegram-ui` — собственные компоненты (Header, ListItem, Avatar, Section, Toggle, ActionButton), соответствующие макетам
- **Новый экран Space Detail**: action-кнопки (изменить, участники, ссылка, удалить) + список событий с дата-бейджами и относительными датами
- **Отдельный экран Members**: список участников с аватарами-инициалами, ролевыми бейджами и инлайн-подтверждением удаления
- **Переработанный Event Detail**: объединённый просмотр + редактирование с полями формы, информацией об авторе, инлайн-подтверждением удаления
- **Экран Space Edit**: редактирование названия пространства (отдельный экран вместо части settings)
- **Система тостов**: всплывающие уведомления об успешных операциях (сохранение, удаление, копирование ссылки)
- **Инлайн-подтверждения**: диалоги удаления прямо в контексте экрана, без модальных оверлеев
- **Аватары с инициалами**: цветовая кодировка по ID, вместо стандартных иконок
- **Визуальная иерархия событий**: плашки с числом и месяцем, относительные даты (сегодня/завтра/послезавтра)
- **Удаление зависимости** от `@telegram-apps/telegram-ui` — весь UI строится на кастомных компонентах

## Capabilities

### New Capabilities
- `miniapp-ui-components`: библиотека кастомных UI-компонентов Mini App (Header, ListItem, Avatar, Section, Divider, Toggle, ActionButton, Toast) с единой дизайн-системой
- `miniapp-space-detail`: экран детального просмотра пространства с action-кнопками и списком событий

### Modified Capabilities
- `mini-app-frontend`: полная переработка всех экранов под новый дизайн — замена Telegram UI на кастомные компоненты, новая навигация, новый визуальный стиль

## Impact

- **Frontend**: полная переработка `mini_app/frontend/src/` — все страницы, компоненты, стили
- **Зависимости**: удаление `@telegram-apps/telegram-ui`, сохранение `@telegram-apps/sdk-react` для инициализации и initData
- **Backend**: без изменений — все существующие API-эндпоинты остаются прежними
- **Роутинг**: переход на history-based навигацию внутри приложения (React Router сохраняется, но маршруты меняются)
- **Темизация**: замена CSS-переменных Telegram UI на собственные токены + поддержка Telegram colorScheme
