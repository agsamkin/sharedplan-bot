## MODIFIED Requirements

### Requirement: Действие "Удалить" с инлайн-подтверждением
Система SHALL при нажатии кнопки "Удалить" отображать инлайн-блок подтверждения. При подтверждении SHALL вызываться API `DELETE /api/spaces/:id`, после успеха -- навигация на `/` и Toast "Пространство удалено". Бэкенд SHALL перед удалением получить название пространства и список всех участников. После успешного удаления бэкенд SHALL отправить Telegram-уведомление каждому участнику пространства (кроме администратора, инициировавшего удаление) о том, что пространство удалено и напоминания из него больше не будут приходить. Ошибки отправки уведомлений SHALL логироваться, но НЕ SHALL влиять на результат API-ответа.

#### Scenario: Удаление пространства
- **WHEN** администратор подтверждает удаление пространства "Семья"
- **THEN** API удаляет пространство, пользователь возвращается на главный экран, появляется Toast "Пространство удалено"

#### Scenario: Уведомление участников при удалении пространства
- **WHEN** администратор "Иван" удаляет пространство "Семья", в котором 3 участника (Иван, Мария, Пётр)
- **THEN** Мария и Пётр получают Telegram-сообщение с названием удалённого пространства, именем удалившего и пояснением что напоминания больше не будут приходить

#### Scenario: Ошибка отправки уведомления не блокирует удаление
- **WHEN** администратор удаляет пространство, но Telegram API недоступен для одного из участников
- **THEN** пространство удаляется успешно, ошибка отправки логируется, остальные участники получают уведомления

#### Scenario: Единственный участник удаляет пространство
- **WHEN** администратор — единственный участник пространства — удаляет его
- **THEN** пространство удаляется, уведомления не отправляются (некому)
