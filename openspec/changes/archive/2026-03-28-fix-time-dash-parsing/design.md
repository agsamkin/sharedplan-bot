## Context

LLM-парсер использует системный промпт (`app/prompts/event_parser.py`) с правилами и few-shot примерами для разбора текста в структурированное событие. Текущее правило для временных интервалов гласит: "Если указан временной интервал (например, «08:00 - 09:30»), в time поставь время начала, а интервал сохрани в названии". Это правило корректно для явных интервалов, но LLM ошибочно применяет его к конструкции "в 14-15", интерпретируя как диапазон 14:00–15:00 вместо времени 14:15.

Проблема исключительно в промпте — код парсера и валидации не требует изменений.

## Goals / Non-Goals

**Goals:**
- Корректный парсинг "в 14-15" / "в 14-30" как времени с минутами (14:15 / 14:30)
- Сохранение корректного парсинга явных интервалов "08:00 - 09:30"
- Сохранение корректного парсинга диапазонов "с 14 до 15"

**Non-Goals:**
- Изменение кода `llm_parser.py` или `ParsedEvent`
- Поддержка новых форматов дат
- Изменение логики retry или валидации

## Decisions

### 1. Добавление явного правила разрешения неоднозначности "число-дефис-число"

**Решение:** Добавить в промпт правило: если после предлога "в" идёт конструкция "ЧЧ-ММ" (где ММ < 60), это время с минутами, а не диапазон.

**Почему:** LLM без явного правила трактует дефис как разделитель диапазона, что является ошибкой для русскоязычного контекста. Добавление эксплицитного правила — наиболее надёжный способ управления поведением LLM.

**Альтернатива:** Убрать правило интервалов полностью — отвергнуто, т.к. интервалы "08:00 - 09:30" парсятся корректно и полезны.

### 2. Уточнение правила временных интервалов

**Решение:** Переформулировать правило интервалов, чтобы оно явно срабатывало только для формата "HH:MM - HH:MM" (с двоеточием и пробелами) или конструкции "с X до Y".

**Почему:** Более точная формулировка снижает вероятность ложного срабатывания на конструкциях вроде "в 14-15".

### 3. Добавление few-shot примеров

**Решение:** Добавить 2 примера:
1. "Завтра обед в 14-15 с Леной" → time: "14:15" (время с минутами, не диапазон)
2. "Встреча с 14 до 15" → time: "14:00", title включает "14:00–15:00" (явный диапазон)

**Почему:** Few-shot примеры — самый эффективный способ направить поведение LLM в неоднозначных случаях.

## Risks / Trade-offs

- **[Риск] LLM может игнорировать правило при нестандартных формулировках** → Митигация: few-shot примеры усиливают правило; temperature=0.1 минимизирует вариативность
- **[Риск] Регрессия в парсинге существующих интервалов** → Митигация: уточнённое правило явно описывает формат интервалов; существующий пример "08:00 - 09:30" сохраняется
