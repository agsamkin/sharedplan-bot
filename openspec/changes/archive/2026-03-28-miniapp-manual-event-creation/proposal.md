## Why

Сейчас события создаются только через бот (текст/голос → LLM-парсинг → подтверждение). Пользователи Mini App могут просматривать и редактировать события, но не могут создать новое прямо из интерфейса. Это вынуждает переключаться из Mini App в чат с ботом. Добавление формы создания события в Mini App завершает CRUD-цикл и делает приложение самодостаточным.

## What Changes

- Новый экран «Новое событие» в Mini App с формой: название (обязательно), дата (обязательно), время (необязательно)
- Кнопка «Добавить» на экране деталей пространства для навигации к форме создания
- Новый POST-эндпоинт в backend API для создания события через Mini App
- При создании — автоматическое создание scheduled_reminders для участников пространства и отправка уведомлений через бот
- Валидация: проверка прошедших дат, проверка конфликтов по времени (±2 часа)

## Capabilities

### New Capabilities
- `miniapp-event-creation`: Экран создания события в Mini App — форма с полями название/дата/время, валидация, сохранение через API, уведомление участников

### Modified Capabilities
- `mini-app-backend`: Добавление POST-эндпоинта `/api/spaces/{space_id}/events` для создания события
- `miniapp-space-detail`: Добавление кнопки «Добавить» в список событий и навигация к экрану создания
- `mini-app-frontend`: Добавление роута `/spaces/:id/events/new` и экрана EventCreate

## Impact

- **Frontend**: Новый компонент EventCreateScreen, обновление SpaceDetailScreen (кнопка «Добавить»), новый роут в React Router
- **Backend API**: Новый эндпоинт POST `/api/spaces/{space_id}/events`, переиспользование event_service для создания события и напоминаний
- **Сервисный слой**: event_service уже поддерживает создание событий — переиспользуется без изменений
- **Уведомления**: Используется существующий механизм notify_space_members
