## Why

В Mini App при создании события поле «Время» помечено как необязательное. Это приводит к тому, что пользователи могут создавать события без времени, что снижает полезность календаря — события без времени не участвуют в проверке конфликтов и не генерируют точные напоминания. В контексте совместного планирования в группе время — критически важная информация.

## What Changes

- Поле «Время» становится обязательным в форме создания события в Mini App
- Убирается пометка «(необязательно)» у лейбла времени
- Кнопка «Создать событие» доступна только при заполнении всех трёх полей: название, дата, время
- Backend API остаётся без изменений — `event_time` по-прежнему опционален для событий, создаваемых через бота (текст/голос)

## Capabilities

### New Capabilities

Нет новых возможностей.

### Modified Capabilities

- `miniapp-event-creation`: Время становится обязательным полем формы (ранее было опциональным)

## Impact

- **Frontend**: `mini_app/frontend/src/pages/EventCreatePage.tsx` — изменение валидации `canSave`, удаление пометки «необязательно»
- **Backend**: Без изменений (API уже поддерживает время, просто фронтенд теперь всегда его отправляет)
- **Бот**: Без изменений (парсинг через LLM и голос по-прежнему допускают события без времени)
