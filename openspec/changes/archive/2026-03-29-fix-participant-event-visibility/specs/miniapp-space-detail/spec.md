## MODIFIED Requirements

### Requirement: Экран Space Detail
Система SHALL отображать экран детального просмотра пространства по маршруту `/spaces/:id`. Экран SHALL содержать: панель action-кнопок сверху, секцию со списком событий. Заголовок Header SHALL показывать имя пространства. Секция событий SHALL использовать `total_count` из API для отображения общего количества событий в заголовке (формат: "События · {total_count}").

#### Scenario: Просмотр пространства с событиями
- **WHEN** пользователь переходит на экран пространства, API возвращает `{ events: [...5], total_count: 5 }`
- **THEN** отображается панель действий и секция "События · 5" со списком из 5 событий

#### Scenario: Пустое пространство
- **WHEN** пространство не содержит событий, API возвращает `{ events: [], total_count: 0 }`
- **THEN** отображается секция "События · 0" с сообщением "Нет предстоящих событий"

#### Scenario: Больше событий чем отображается
- **WHEN** API возвращает `{ events: [...50], total_count: 75 }`
- **THEN** секция показывает "События · 75", список содержит 50 событий
