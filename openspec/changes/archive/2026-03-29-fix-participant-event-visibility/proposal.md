## Why

Участник пространства не видит события, созданные владельцем (администратором) этого пространства. Это критический баг — без видимости чужих событий теряется весь смысл совместного планирования.

## What Changes

- Добавить диагностическое логирование в API-эндпоинт `GET /api/spaces/{space_id}/events` для отладки: какой `user_id` запрашивает, какой `space_id`, сколько событий найдено
- Убрать ограничение `limit=10` в `get_upcoming_events` или сделать его параметром запроса с пагинацией, чтобы все события были доступны
- Добавить в API-ответ `/api/spaces/{space_id}/events` полное количество событий (`total_count`) для диагностики расхождений между тем, что видит владелец и участник
- Проверить и при необходимости исправить обработку ошибок на фронтенде — если API возвращает ошибку, пользователь должен видеть информативное сообщение, а не пустой список
