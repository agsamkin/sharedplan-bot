## MODIFIED Requirements

### Requirement: Создание пространства через /newspace с FSM

Команда `/newspace` ДОЛЖНА запускать FSM-флоу создания пространства. Бот ДОЛЖЕН запросить название, валидировать его (1-255 символов), сгенерировать уникальный invite-код через `secrets.token_urlsafe(6)`, создать записи в `spaces` и `user_spaces` (role='admin') в одной транзакции, и ответить подтверждением с invite-ссылкой.

Сообщение об успешном создании ДОЛЖНО быть дружелюбным и содержать чёткую инструкцию по приглашению участников.

#### Scenario: Успешное создание пространства

- **WHEN** пользователь отправляет `/newspace` и вводит название "Семья"
- **THEN** бот создаёт пространство и отвечает: `✅ Пространство «Семья» создано!\n\n🔗 Ссылка для приглашения:\n{invite_link}\n\nОтправь эту ссылку тем, кого хочешь добавить.`

#### Scenario: Название пространства слишком длинное

- **WHEN** пользователь вводит название длиннее 255 символов
- **THEN** бот отвечает: «Слишком длинное название. Попробуй покороче (до 255 символов).»

#### Scenario: Пустое название пространства

- **WHEN** пользователь отправляет пустое сообщение или пробелы в ответ на запрос названия
- **THEN** бот отвечает: «Название не может быть пустым. Введи название пространства:»

#### Scenario: FSM-состояние при /newspace

- **WHEN** пользователь отправляет `/newspace`
- **THEN** бот устанавливает FSM-состояние `CreateSpace.waiting_for_name` и отправляет сообщение «Как назовём пространство?»

### Requirement: Список пространств через /spaces

Команда `/spaces` ДОЛЖНА показывать список пространств пользователя с названием, ролью и количеством участников.

#### Scenario: Пользователь состоит в нескольких пространствах

- **WHEN** пользователь, состоящий в пространствах "Семья" (admin, 3 участника) и "Работа" (member, 5 участников), отправляет `/spaces`
- **THEN** бот отвечает нумерованным списком с эмодзи-индикаторами ролей: `📂 Твои пространства:\n\n1. 👑 Семья (3 участника)\n2. 👤 Работа (5 участников)`

#### Scenario: Пользователь не состоит ни в одном пространстве

- **WHEN** пользователь, не состоящий ни в одном пространстве, отправляет `/spaces`
- **THEN** бот отвечает: «У тебя пока нет пространств. Создай первое через /newspace!»
