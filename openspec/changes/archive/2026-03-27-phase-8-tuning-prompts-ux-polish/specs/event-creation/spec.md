## MODIFIED Requirements

### Requirement: Карточка подтверждения события

После парсинга и выбора пространства бот ДОЛЖЕН отправить карточку подтверждения с данными события и тремя inline-кнопками.

При наличии транскрипции (голосовой ввод) карточка ДОЛЖНА начинаться со строки `🎤 Распознано: «{transcript}»` перед данными события.

Дата в карточке ДОЛЖНА включать день недели в формате «{день} {месяц} {год}, {день_недели}» (например, «28 марта 2026, суббота»). Это помогает пользователю верифицировать корректность распознавания относительных дат.

#### Scenario: Событие с временем
- **WHEN** парсинг вернул событие с датой 2026-03-28 (суббота) и временем 19:00
- **THEN** бот отправляет карточку формата: `📌 Событие:\n📝 {title}\n📅 28 марта 2026, суббота\n⏰ 19:00\n\nОпубликовать?` с кнопками `[✅ Да]` `[✏️ Изменить]` `[❌ Отмена]`

#### Scenario: Событие без времени
- **WHEN** парсинг вернул событие без времени (time=None)
- **THEN** карточка отображается без строки ⏰, дата включает день недели

#### Scenario: Голосовой ввод — карточка с транскрипцией
- **WHEN** событие создано из голосового сообщения и транскрипция присутствует в FSM-данных
- **THEN** карточка начинается с `🎤 Распознано: «{transcript}»\n\n`, затем стандартный формат `📌 Событие:...`

#### Scenario: Текстовый ввод — карточка без транскрипции
- **WHEN** событие создано из текстового сообщения (транскрипция отсутствует)
- **THEN** карточка отображается в стандартном формате без строки 🎤

### Requirement: Уведомление участников пространства

После сохранения события все участники пространства (кроме создателя) ДОЛЖНЫ получить уведомление.

Дата в уведомлении ДОЛЖНА использовать расширенный формат относительных дат: «сегодня», «завтра», «послезавтра», день недели для ближайших 6 дней (в винительном падеже: «в среду», «в пятницу»), числовой формат с месяцем для остальных.

#### Scenario: Уведомление при публикации — событие послезавтра
- **WHEN** событие подтверждено с датой послезавтра в 14:00
- **THEN** участники получают уведомление: `📅 Новое событие в «{space_name}»!\n\n📝 {title}\n📅 Послезавтра\n⏰ 14:00\n👤 Добавил: {creator_name}`

#### Scenario: Уведомление при публикации — событие в среду
- **WHEN** событие подтверждено на ближайшую среду
- **THEN** участники получают уведомление с датой «В среду» вместо числового формата

#### Scenario: Событие без времени — уведомление без ⏰
- **WHEN** событие без времени подтверждено
- **THEN** уведомление отправляется без строки ⏰

#### Scenario: Ошибка отправки одному участнику
- **WHEN** отправка уведомления одному участнику не удалась (пользователь заблокировал бота)
- **THEN** ошибка логируется, уведомления остальным участникам отправляются нормально

#### Scenario: Создатель единственный участник
- **WHEN** событие подтверждено, но создатель — единственный участник пространства
- **THEN** уведомления никому не отправляются (нет других участников)

### Requirement: Парсинг структурированного текста (заглушка)

Заглушка `parse_event_text()` ЗАМЕНЯЕТСЯ на вызов `llm_parser.parse_event()`. Хендлер `bot/handlers/event.py` ДОЛЖЕН передавать текст пользователя в LLM-парсер вместо ручного парсинга по формату.

При ошибке парсинга хендлер ДОЛЖЕН отображать дружелюбное сообщение в зависимости от типа ошибки, без технических деталей.

#### Scenario: Пользователь пишет естественным языком
- **WHEN** пользователь отправляет «Ужин с родителями завтра в 19:00»
- **THEN** текст передаётся в `llm_parser.parse_event()`, результат отображается в карточке подтверждения

#### Scenario: LLM не смог распознать событие
- **WHEN** `llm_parser.parse_event()` выбрасывает `ParseError` с типом `invalid_json`
- **THEN** бот отвечает: «Не удалось распознать событие. Попробуй описать его проще, например: «Ужин завтра в 19:00».»

#### Scenario: Таймаут или ошибка сети
- **WHEN** `llm_parser.parse_event()` выбрасывает `ParseError` с типом `timeout` или `network`
- **THEN** бот отвечает: «Не удалось связаться с сервисом. Попробуй ещё раз через пару секунд.»

#### Scenario: Rate limit исчерпан
- **WHEN** `llm_parser.parse_event()` выбрасывает `ParseError` с типом `rate_limit`
- **THEN** бот отвечает: «Сервис временно перегружен. Попробуй через минуту.»

#### Scenario: Индикатор набора текста
- **WHEN** пользователь отправляет текстовое сообщение для создания события
- **THEN** бот отправляет `chat_action: typing` перед вызовом LLM
