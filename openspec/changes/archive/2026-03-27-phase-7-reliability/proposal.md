## Why

Бот реализован end-to-end (Фазы 1–6), но ряд edge cases, валидаций и отказоустойчивости пропущен: событие можно создать в прошлом без предупреждения, текст события не ограничен по длине, first_name пользователя не обновляется, при старте нет проверки связности с БД и валидности токена. Логирование есть, но неоднородно — команды пользователей не логируются, формат не стандартизирован. Фаза 7 закрывает эти пробелы перед финальной полировкой (Фаза 8).

## What Changes

- **Валидация ввода**: ограничение текста события (1–1000 символов), валидация даты (не более 2 лет в будущее), предупреждение о дате в прошлом с inline-подтверждением «Всё равно создать?»
- **Обновление first_name**: middleware обновляет `first_name` и `username` пользователя при каждом взаимодействии
- **Проверки при старте**: валидация обязательных env-переменных (уже через pydantic), проверка связности с PostgreSQL, проверка Telegram-токена через `getMe` — до запуска polling
- **Стандартизация логирования**: единый формат для всех LLM/Nexara/Telegram-вызовов, логирование каждой пользовательской команды (user_id, command, space_id), логирование всех ошибок отправки Telegram (user_id, error_type)
- **Ужесточение edge cases**: обработка пустой транскрипции от Nexara (уже есть, проверить), обработка заблокированного бота (уже частично есть, унифицировать), невалидный invite-код (уже есть)

## Capabilities

### New Capabilities

- `input-validation`: Централизованные правила валидации пользовательского ввода — длина текста события, диапазон дат, предупреждение о прошедшей дате с inline-подтверждением
- `startup-checks`: Проверка конфигурации и связности при старте — PostgreSQL connectivity, Telegram token (getMe), все обязательные env-переменные

### Modified Capabilities

- `event-creation`: Добавляется предупреждение о дате в прошлом с inline-кнопкой «Всё равно создать?», валидация длины текста (1–1000), валидация диапазона даты (макс. 2 года)
- `telegram-bot-core`: Добавляется middleware обновления first_name/username при каждом взаимодействии, стандартизированное логирование команд (user_id, command, space_id)
- `llm-event-parser`: Добавляется валидация диапазона даты в Pydantic-схеме ParsedEvent (не более 2 лет в будущее)

## Impact

- **Handlers** (`app/bot/handlers/event.py`, `voice.py`): добавление валидации длины текста и проверки даты в прошлом
- **Callbacks** (`app/bot/callbacks/event_confirm.py`): поддержка подтверждения события с прошедшей датой
- **Services** (`app/services/llm_parser.py`): валидация диапазона даты в ParsedEvent
- **Middleware** (`app/bot/middlewares/`): новый middleware обновления профиля пользователя
- **Main** (`app/main.py`): добавление startup-проверок перед polling
- **Keyboards** (`app/bot/keyboards/`): новая клавиатура для подтверждения прошедшей даты
- **Логирование**: все модули — стандартизация формата, добавление логирования команд
