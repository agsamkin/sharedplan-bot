## Why

При добавлении события пользователь может не заметить, что на это же время (или в пределах 2 часов) уже запланировано другое событие в том же пространстве. Это приводит к конфликтам расписания, которые обнаруживаются только когда уже поздно. Предупреждение на этапе подтверждения позволит пользователю осознанно принять решение.

## What Changes

- Добавляется проверка пересечения событий по времени при создании нового события: если в том же пространстве на ту же дату есть события в окне +-2 часа, пользователю показывается предупреждение с перечнем конфликтующих событий
- Предупреждение не блокирует создание — пользователь может подтвердить создание несмотря на конфликт
- Проверка работает как для текстового, так и для голосового ввода
- Проверка работает и при создании события через Mini App

## Capabilities

### New Capabilities
- `event-time-conflict-check`: Проверка пересечений по времени при создании события — сервисная логика поиска конфликтующих событий в пространстве и отображение предупреждения пользователю

### Modified Capabilities
- `event-creation`: Интеграция предупреждения о конфликте времени в флоу подтверждения события (аналогично существующему предупреждению о прошедшей дате)

## Impact

- **Сервисный слой**: новая функция в `event_service.py` для поиска конфликтующих событий
- **Хендлеры бота**: модификация `process_parsed_event()` и callback-ов подтверждения для показа предупреждения
- **Mini App backend**: модификация API создания событий для возврата информации о конфликтах
- **Форматирование**: новая функция форматирования предупреждения о конфликтах в `formatting.py`
- **БД**: запрос к существующей таблице `events` — новых таблиц и миграций не требуется
