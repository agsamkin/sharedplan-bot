## Why

Сейчас бот требует наличия ключей `OPENROUTER_API_KEY` и `NEXARA_API_KEY` для запуска — без них приложение падает на этапе валидации конфигурации. Это мешает деплою в средах, где не все внешние сервисы доступны или оплачены, и не позволяет использовать бот в ограниченном режиме (только ручное создание событий через Mini App).

## What Changes

- **Ключи API становятся опциональными**: `OPENROUTER_API_KEY`, `OPENROUTER_MODEL` и `NEXARA_API_KEY` больше не обязательны в конфигурации. Приложение стартует без них.
- **Graceful degradation для голосового ввода**: при отсутствии `NEXARA_API_KEY` бот отвечает на голосовые сообщения предупреждением, что функция голосовых сообщений отключена.
- **Graceful degradation для текстового парсинга**: при отсутствии `OPENROUTER_API_KEY` бот отвечает на текстовые сообщения предупреждением, что автоматическое распознавание событий отключено и предлагает использовать Mini App для ручного создания.
- **Логирование при старте**: при запуске бот логирует информацию о доступных/недоступных функциях на основе наличия ключей.

## Capabilities

### New Capabilities

Нет новых capabilities.

### Modified Capabilities

- `app-config`: Ключи `OPENROUTER_API_KEY`, `OPENROUTER_MODEL` и `NEXARA_API_KEY` становятся опциональными (Optional с default None)
- `speech-to-text`: Добавляется проверка доступности сервиса — новый тип ошибки `service_disabled` при отсутствии ключа
- `llm-event-parser`: Добавляется проверка доступности сервиса — новый тип ошибки `service_disabled` при отсутствии ключа
- `voice-event-handler`: Обработка случая недоступности STT — отправка предупреждения пользователю вместо попытки транскрибации
- `event-creation`: Обработка случая недоступности LLM — отправка предупреждения с рекомендацией использовать Mini App
- `startup-checks`: Убрать жёсткое требование API-ключей, добавить информационное логирование о доступных функциях

## Impact

- **Конфигурация** (`app/config.py`): поля становятся Optional
- **Сервисы** (`app/services/speech_to_text.py`, `app/services/llm_parser.py`): ранний выход с ошибкой `service_disabled` при отсутствии ключа
- **Хэндлеры** (`app/bot/handlers/voice.py`, `app/bot/handlers/event.py`): обработка нового типа ошибки с пользовательским сообщением
- **Форматирование** (`app/bot/formatting.py`): новые сообщения для отключённых функций
- **Старт** (`app/main.py`): логирование статуса функций вместо падения
- **Docker/env** (`.env.example`): пометка ключей как опциональных
