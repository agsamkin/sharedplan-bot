## Why

Бот предназначен для личного/семейного использования. Сейчас любой пользователь Telegram может начать взаимодействие с ботом через `/start`, создавать пространства и события. Необходимо ограничить доступ: ботом могут пользоваться только владелец и те, кого он пригласил через пригласительную ссылку пространства.

## What Changes

- Добавляется глобальный контроль доступа на уровне бота (middleware), блокирующий все взаимодействия от неавторизованных пользователей
- Владелец бота определяется через конфигурацию (`OWNER_TELEGRAM_ID`) и всегда имеет доступ
- Пользователь получает доступ к боту при вступлении в пространство по пригласительной ссылке (существующий механизм deep link `?start=join_<code>`)
- При исключении из всех пространств (kick) или удалении всех пространств пользователь теряет доступ к боту
- Неавторизованным пользователям показывается сообщение о том, что бот приватный
- Команда `/start` без параметров доступна для владельца; для остальных — только `/start join_<code>` (deep link для вступления)

## Capabilities

### New Capabilities
- `bot-access-control`: Middleware контроля доступа на уровне бота — проверка авторизации пользователя на каждом входящем update, блокировка неавторизованных, управление статусом доступа

### Modified Capabilities
- `app-config`: Новый обязательный параметр `OWNER_TELEGRAM_ID` (Telegram ID владельца бота)
- `space-joining`: Вступление в пространство по инвайт-ссылке теперь также даёт доступ к боту
- `space-management`: Исключение пользователя (kick) и удаление пространства могут привести к отзыву доступа к боту, если пользователь больше не состоит ни в одном пространстве

## Impact

- **Middleware**: Новый middleware, выполняющийся до всех остальных (кроме `DbSessionMiddleware`)
- **БД**: Изменение таблицы `users` (новая колонка для статуса доступа) + миграция
- **Конфигурация**: Новый обязательный env var `OWNER_TELEGRAM_ID`
- **Handlers**: Модификация `start.py` (логика для неавторизованных), `space.py` (отзыв доступа при kick/delete)
- **Services**: Новый сервис или расширение существующих для управления статусом доступа
