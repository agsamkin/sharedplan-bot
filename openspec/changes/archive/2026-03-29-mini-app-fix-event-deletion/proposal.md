## Why

Удаление события в Mini App не работает. После нажатия «Удалить событие» → «Удалить» в ConfirmInline ничего не происходит — событие остаётся в списке. Причина: API-клиент (`client.ts`) отправляет DELETE-запрос с заголовком `Content-Type: application/json` без тела, что может вызывать проблемы в некоторых окружениях (Telegram WebView). Кроме того, после успешного удаления `navigate(-1)` возвращает на SpaceDetailPage, но в Telegram WebView компонент может не перемонтироваться (bfcache), и данные не обновляются — пользователь видит удалённое событие в списке.

## What Changes

- Исправить API-клиент: не отправлять заголовок `Content-Type: application/json` для запросов без тела (DELETE)
- Добавить индикатор загрузки (deleting state) на кнопку удаления, чтобы предотвратить повторные нажатия
- Добавить обработку ошибок с отображением toast-уведомления при неудачном удалении
- После успешного удаления использовать навигацию с заменой в истории вместо `navigate(-1)`, чтобы гарантировать перемонтирование SpaceDetailPage

### Modified Capabilities

## Impact

- **Frontend API-клиент** (`mini_app/frontend/src/api/client.ts`): условная установка `Content-Type` только при наличии тела запроса
- **Страница события** (`mini_app/frontend/src/pages/EventDetailPage.tsx`): состояние `deleting`, блокировка кнопки, toast при ошибке, изменение навигации после удаления
- **Backend**: изменений не требуется, DELETE-эндпоинт работает корректно
