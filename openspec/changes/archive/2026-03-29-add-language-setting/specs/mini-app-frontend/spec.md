## MODIFIED Requirements

### Requirement: Экран списка пространств
Отображает пространства пользователя в Section, каждое как ListItem с Avatar, названием, подписью "N участн. · M событий", ChevronRight. Ниже — Section "Настройки" с ListItem "Напоминания" и ListItem "Язык" (иконка IconGlobe, подзаголовок — название текущего языка, например "Русский" или "English"). При загрузке — loading-состояние, при ошибке — error с кнопкой повтора, при пустом списке — empty-состояние. Хедер справа — кнопка "+" (IconPlus) для создания пространства.

#### Scenario: Отображение настройки языка
- **WHEN** пользователь открывает экран пространств
- **THEN** в секции "Настройки" отображается ListItem "Язык" с иконкой глобуса и подзаголовком — названием текущего языка

#### Scenario: Переход к экрану языка
- **WHEN** пользователь нажимает на ListItem "Язык"
- **THEN** открывается экран выбора языка (`/settings/language`)

### Requirement: Навигация и BackButton
React Router с маршрутами: `/`, `/spaces/new`, `/spaces/:id`, `/spaces/:id/edit`, `/spaces/:id/members`, `/spaces/:id/events/new`, `/events/:id`, `/settings/reminders`, `/settings/language`. Telegram BackButton скрыт на `/`, показан на остальных, нажатие — `navigate(-1)`.

#### Scenario: Навигация на экран языка
- **WHEN** пользователь переходит на `/settings/language`
- **THEN** отображается экран выбора языка, BackButton видна

#### Scenario: Возврат с экрана языка
- **WHEN** пользователь нажимает BackButton на `/settings/language`
- **THEN** возвращается на `/`

## ADDED Requirements

### Requirement: Экран выбора языка
Маршрут `/settings/language`. Хедер с заголовком "Язык" / "Language". Section с заголовком "Язык интерфейса" / "Interface language". Два варианта:
- "Русский" — при выбранном `ru` отображается жирным шрифтом (fontWeight 600) с галочкой (SVG checkmark, цвет #378ADD)
- "English" — при выбранном `en` отображается жирным шрифтом с галочкой

При нажатии на вариант — вызов `PUT /api/user/language`, немедленное переключение UI на выбранный язык. Все строки приложения ДОЛЖНЫ мгновенно обновиться без перезагрузки.

#### Scenario: Отображение текущего языка
- **WHEN** пользователь с `language = "ru"` открывает экран
- **THEN** "Русский" отображается жирным с галочкой, "English" — обычным без галочки

#### Scenario: Переключение на английский
- **WHEN** пользователь нажимает "English"
- **THEN** UI переключается на английский, отправляется `PUT /api/user/language` с `{"language": "en"}`, "English" становится жирным с галочкой

#### Scenario: Ошибка при сохранении языка
- **WHEN** API возвращает ошибку при `PUT /api/user/language`
- **THEN** UI откатывается к предыдущему языку

### Requirement: Все строки Mini App через i18n
Все текстовые строки во всех компонентах и страницах Mini App ДОЛЖНЫ загружаться из i18n-словаря через хук `useTranslation()`. Захардкоженные русские строки ДОЛЖНЫ быть полностью заменены. Словари ДОЛЖНЫ содержать переводы для всех UI-элементов: заголовки, кнопки, подписи, placeholder-ы, сообщения об ошибках, toast-сообщения, лейблы форм, пустые состояния.

#### Scenario: Все страницы на английском
- **WHEN** пользователь с `language = "en"` открывает любую страницу Mini App
- **THEN** все строки отображаются на английском языке

#### Scenario: Все страницы на русском
- **WHEN** пользователь с `language = "ru"` открывает любую страницу Mini App
- **THEN** все строки отображаются на русском языке

### Requirement: Загрузка языка при инициализации
При старте Mini App ДОЛЖЕН загружать язык пользователя через `GET /api/user/language` и применять его до отображения контента. До получения языка показывается loading-состояние.

#### Scenario: Загрузка языка при старте
- **WHEN** Mini App запускается
- **THEN** отправляется `GET /api/user/language`, полученный язык применяется, затем отображается контент
