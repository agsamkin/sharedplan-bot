## Context

Бот в текущем состоянии регистрирует 6 команд: start, help, newspace, spaces, events, app. Команды newspace, spaces, space_info, events, reminders дублируют функциональность Mini App. При этом Mini App уже спроектирован с экранами управления пространствами, событиями и напоминаниями (макет в `docs/shared-plan-miniapp.jsx`).

Текущая архитектура:
- Хэндлеры команд: `space.py` (newspace, spaces, space_info), `events_list.py` (events), `reminders.py` (reminders)
- FSM: `CreateSpace` для пошагового создания пространства через бота
- Callback-хэндлеры: `space_select_cb.py` для inline-выбора пространства
- Сервисный слой: `space_service.py` — уже изолирован от Telegram, может использоваться из Mini App API
- Mini App Backend: `mini_app/backend/app.py` — aiohttp-приложение, запускается вместе с ботом

## Goals / Non-Goals

**Goals:**
- Упростить набор команд бота до /start, /help, /privacy
- Добавить команду /privacy (ссылка на Telegram Privacy Policy)
- Реализовать экран создания пространства в Mini App (по макету)
- Добавить API-эндпоинт POST /api/spaces для создания пространства
- Обновить /start и /help для направления пользователей в Mini App

**Non-Goals:**
- Изменение логики работы событий, напоминаний или invite-flow
- Рефакторинг Mini App frontend (кроме добавления экрана создания пространства)
- Изменение схемы БД
- Миграция существующих пользовательских данных

## Decisions

### 1. Удаление хэндлеров vs. перенаправление

**Решение**: Полностью удалить хэндлеры удалённых команд, не оставлять «заглушки» с перенаправлением.

**Почему**: Команды не зарегистрированы в BotFather — пользователь не увидит их в меню. Если кто-то наберёт их вручную, aiogram просто проигнорирует. Заглушки создают ненужную поддержку кода.

**Альтернатива**: Оставить хэндлеры-заглушки с сообщением «используйте Mini App» — отвергнуто, т.к. усложняет кодовую базу без пользы.

### 2. Размещение экрана создания пространства

**Решение**: Экран `spaceCreate` в Mini App, доступный по кнопке "+" в хедере списка пространств (как в макете). Навигация: spaces → spaceCreate → (создание) → spaces.

**Почему**: Следуем макету. Кнопка "+" в хедере — стандартный паттерн мобильных приложений.

### 3. API-эндпоинт создания пространства

**Решение**: `POST /api/spaces` с телом `{ "name": "string" }`. Использует существующий `space_service.create_space()`. Возвращает созданное пространство с invite_code.

**Почему**: Сервисный слой уже реализован и протестирован. API просто маршрутизирует запрос к сервису. Соответствует REST-конвенции существующих эндпоинтов.

### 4. Обновление /start

**Решение**: Приветственное сообщение с описанием бота и кнопкой WebApp для открытия Mini App. Если пользователь приходит по deep link (join_*), сначала обрабатываем присоединение, затем показываем приветствие.

**Почему**: /start — единственная точка входа для новых пользователей. Должен направлять в Mini App.

### 5. Команда /privacy

**Решение**: Простой хэндлер, отправляющий сообщение со ссылкой на https://telegram.org/privacy. Без inline-кнопок — просто текст со ссылкой.

**Почему**: Минимальная реализация, соответствует требованиям Telegram для ботов.

## Risks / Trade-offs

- **Пользователи привыкли к командам** → Миграция: обновлённый /help ясно направляет в Mini App. Со временем пользователи адаптируются.
- **Mini App недоступен (нет WebApp)** → Старые клиенты Telegram без поддержки WebApp не смогут создавать пространства. Митигация: join по invite-ссылке (/start join_*) остаётся в боте.
- **Порядок удаления** → Сначала добавить функциональность в Mini App, затем удалять команды бота. Обеспечивает отсутствие периода, когда функциональность недоступна.
