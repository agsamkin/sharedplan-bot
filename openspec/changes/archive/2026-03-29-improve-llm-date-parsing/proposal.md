## Why

LLM некорректно вычисляет даты из относительных выражений ("завтра", "в понедельник"). Конкретный баг: запрос "Завтра обед в 14" при текущей дате 2026-03-29 вернул 28 марта (вчерашнюю дату). Причина — few-shot примеры в промпте содержат хардкоженные даты (2026-03-27, пятница), которые конфликтуют с реальной текущей датой и сбивают модель. Кроме того, промпт недостаточно явно акцентирует приоритет актуальной даты над примерами.

## What Changes

- Сделать few-shot примеры динамическими — вычислять даты в примерах относительно текущей даты, чтобы они всегда были консистентны с контекстом "Сегодня: ..."
- Усилить системный промпт: добавить явное правило приоритета актуальной даты и предупреждение не использовать даты из примеров напрямую
- Расширить набор few-shot примеров для лучшего покрытия граничных случаев (выходные, конец месяца, конец года)
- Добавить пост-валидацию: если LLM вернул дату в прошлом для относительного выражения ("завтра", "в понедельник"), логировать предупреждение
