## MODIFIED Requirements

### Requirement: Команда /events — ссылка на Mini App

Команда `/events` ДОЛЖНА направлять пользователя в Mini App для просмотра событий. Если пользователь состоит в пространствах, бот ДОЛЖЕН отправить inline-кнопку «Открыть события» со ссылкой на Mini App.

#### Scenario: Пользователь с пространствами отправляет /events
- **WHEN** пользователь, состоящий в пространствах, отправляет `/events`
- **THEN** бот отвечает сообщением «Открой приложение для просмотра и управления событиями:» с inline-кнопкой WebApp «📅 Открыть события»

#### Scenario: Пользователь не в пространстве
- **WHEN** пользователь, не состоящий ни в одном пространстве, отправляет `/events`
- **THEN** бот отвечает: «Сначала создай пространство через /newspace»

## REMOVED Requirements

### Requirement: Отображение списка событий в боте
**Reason**: Список событий перенесён в Mini App, где доступен полноценный UI со скроллом и управлением.
**Migration**: Команда `/events` теперь открывает Mini App.

### Requirement: Относительные даты для ближайших событий
**Reason**: Логика относительных дат переносится в Mini App фронтенд.
**Migration**: Реализована в `mini-app-frontend` спеке.

### Requirement: Отображение событий без времени
**Reason**: Логика отображения переносится в Mini App фронтенд.
**Migration**: Реализована в `mini-app-frontend` спеке.

### Requirement: Сортировка событий
**Reason**: Сортировка переносится в API бэкенда Mini App.
**Migration**: Реализована в `mini-app-backend` спеке (GET /api/spaces/:id/events).

### Requirement: Только будущие события
**Reason**: Фильтрация переносится в API бэкенда Mini App.
**Migration**: Реализована в `mini-app-backend` спеке.
