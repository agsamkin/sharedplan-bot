## MODIFIED Requirements

### Requirement: Карточка управления событием

Владелец события ДОЛЖЕН иметь возможность управлять своим событием через Telegram Mini App. В списке `/events` бота кнопка «Управление» заменяется на ссылку в Mini App.

#### Scenario: Открытие управления событием
- **WHEN** владелец хочет управлять своим событием
- **THEN** он открывает Mini App, где доступно редактирование и удаление

#### Scenario: Кнопка управления видна только владельцу
- **WHEN** пользователь просматривает события в Mini App
- **THEN** кнопки редактирования и удаления отображаются только для событий, созданных этим пользователем

#### Scenario: Событие уже удалено
- **WHEN** пользователь пытается открыть редактирование события, которое было удалено
- **THEN** Mini App показывает сообщение «Событие не найдено» и обновляет список

### Requirement: Удаление события владельцем

Владелец события ДОЛЖЕН иметь возможность удалить своё событие с подтверждением действия через Mini App.

#### Scenario: Запрос подтверждения удаления
- **WHEN** владелец нажимает кнопку «Удалить» в Mini App
- **THEN** Mini App показывает диалог «Удалить событие «{title}»? Это действие нельзя отменить.» с кнопками «Удалить» и «Отмена»

#### Scenario: Подтверждение удаления
- **WHEN** владелец подтверждает удаление в Mini App
- **THEN** событие удаляется через API, связанные напоминания удаляются каскадно, участники пространства (кроме владельца) получают уведомление через бота, список обновляется

#### Scenario: Отмена удаления
- **WHEN** владелец нажимает «Отмена» в диалоге подтверждения
- **THEN** диалог закрывается, событие остаётся без изменений

#### Scenario: Удаление несуществующего события
- **WHEN** владелец подтверждает удаление события, которое уже было удалено
- **THEN** API возвращает 404, Mini App показывает сообщение «Событие не найдено»

### Requirement: Редактирование названия события

Владелец события ДОЛЖЕН иметь возможность изменить название своего события через форму в Mini App.

#### Scenario: Редактирование названия
- **WHEN** владелец изменяет название в форме редактирования Mini App
- **THEN** новое название отправляется через `PUT /api/events/:id`, участники получают уведомление

#### Scenario: Слишком длинное название
- **WHEN** владелец вводит название длиннее 500 символов
- **THEN** форма показывает ошибку валидации «Название слишком длинное (макс. 500 символов)»

### Requirement: Редактирование даты события

Владелец события ДОЛЖЕН иметь возможность изменить дату своего события через date picker в Mini App.

#### Scenario: Успешное изменение даты
- **WHEN** владелец выбирает новую дату в date picker и сохраняет
- **THEN** дата обновляется через API, неотправленные напоминания пересоздаются, участники получают уведомление

### Requirement: Редактирование времени события

Владелец события ДОЛЖЕН иметь возможность изменить или убрать время своего события через Mini App.

#### Scenario: Успешное изменение времени
- **WHEN** владелец выбирает новое время в time picker и сохраняет
- **THEN** время обновляется через API, неотправленные напоминания пересоздаются, участники получают уведомление

#### Scenario: Удаление времени (событие на весь день)
- **WHEN** владелец отмечает чекбокс «Весь день»
- **THEN** время события устанавливается в NULL через API, напоминания пересоздаются

### Requirement: Уведомления участников при изменении/удалении

Все участники пространства (кроме владельца события) ДОЛЖНЫ получать уведомление через Telegram бота при редактировании или удалении события из Mini App.

#### Scenario: Уведомление при редактировании
- **WHEN** владелец изменяет любое поле события через Mini App
- **THEN** всем участникам пространства (кроме владельца) отправляется сообщение через бота с указанием что изменилось

#### Scenario: Уведомление при удалении
- **WHEN** владелец удаляет событие через Mini App
- **THEN** всем участникам пространства (кроме владельца) отправляется сообщение через бота

#### Scenario: Ошибка отправки уведомления
- **WHEN** уведомление не удаётся отправить одному из участников
- **THEN** ошибка логируется, остальные уведомления продолжают отправляться

### Requirement: Пересоздание напоминаний при изменении даты/времени

При изменении даты или времени события через Mini App ВСЕ неотправленные напоминания ДОЛЖНЫ быть пересозданы с учётом новых значений.

#### Scenario: Пересоздание напоминаний после изменения даты
- **WHEN** владелец изменяет дату события через Mini App
- **THEN** все напоминания с `sent=FALSE` удаляются, новые напоминания создаются по настройкам каждого участника пространства на основе новой даты

#### Scenario: Пересоздание напоминаний после изменения времени
- **WHEN** владелец изменяет время события через Mini App
- **THEN** все напоминания с `sent=FALSE` удаляются, новые напоминания создаются по настройкам каждого участника пространства на основе нового времени

#### Scenario: Напоминания не пересоздаются при изменении названия
- **WHEN** владелец изменяет только название события
- **THEN** напоминания остаются без изменений

### Requirement: Только будущие события доступны для управления

Кнопки редактирования и удаления ДОЛЖНЫ отображаться только для будущих событий в Mini App.

#### Scenario: Будущее событие
- **WHEN** событие с `event_date >= текущая дата`
- **THEN** кнопки редактирования и удаления отображаются для владельца

#### Scenario: Прошедшее событие
- **WHEN** событие с `event_date < текущая дата`
- **THEN** кнопки редактирования и удаления НЕ отображаются
