## ADDED Requirements

### Requirement: Присоединение по deep link

При переходе по invite-ссылке `https://t.me/<bot_username>?start=join_<invite_code>` бот ДОЛЖЕН распознать deep link payload, найти пространство по invite_code, добавить пользователя как member и уведомить текущих участников.

#### Scenario: Успешное присоединение нового участника

- **WHEN** пользователь переходит по invite-ссылке с валидным invite_code и ещё не состоит в этом пространстве
- **THEN** создаётся запись в `user_spaces` с role='member', пользователь получает сообщение «Ты присоединился к пространству {name}!», все текущие участники получают уведомление «{first_name} присоединился к {name}»

#### Scenario: Пользователь уже состоит в пространстве

- **WHEN** пользователь переходит по invite-ссылке пространства, в котором уже состоит
- **THEN** бот отвечает сообщением «Ты уже состоишь в этом пространстве»

#### Scenario: Невалидный invite-код

- **WHEN** пользователь переходит по ссылке с несуществующим invite_code
- **THEN** бот отвечает сообщением «Ссылка недействительна или пространство удалено»

#### Scenario: Upsert пользователя при join

- **WHEN** пользователь, отсутствующий в таблице `users`, переходит по invite-ссылке
- **THEN** сначала создаётся запись в `users` (upsert), затем пользователь добавляется в пространство

### Requirement: Уведомление участников о новом члене

При присоединении нового участника система ДОЛЖНА отправить уведомление каждому текущему участнику пространства (кроме присоединившегося). Каждая отправка ДОЛЖНА быть обёрнута в try/except — ошибка отправки одному участнику НЕ ДОЛЖНА прерывать уведомление остальных.

#### Scenario: Все участники получают уведомление

- **WHEN** пользователь «Анна» присоединяется к пространству с 3 участниками
- **THEN** каждому из 3 участников отправляется сообщение «Анна присоединился(ась) к {space_name}»

#### Scenario: Ошибка отправки одному участнику

- **WHEN** один из участников заблокировал бота
- **THEN** ошибка отправки этому участнику логируется, но уведомления остальным отправляются успешно

### Requirement: Формирование invite-ссылки

Invite-ссылка ДОЛЖНА формироваться в формате `https://t.me/{bot_username}?start=join_{invite_code}`. `bot_username` ДОЛЖЕН получаться через Telegram Bot API (`bot.get_me()`).

#### Scenario: Корректная invite-ссылка

- **WHEN** создаётся пространство с invite_code «abc12345»
- **THEN** invite-ссылка имеет формат `https://t.me/MyBot?start=join_abc12345` (где MyBot — username бота)
