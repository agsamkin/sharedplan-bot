## Context

В проекте 7 тестовых модулей с 60+ тестами. Тесты используют `pytest` и `pytest-asyncio`, но эти пакеты не указаны в зависимостях. Конфигурация pytest отсутствует — используются дефолтные настройки. Измерение покрытия не настроено.

Текущая структура: единственный `requirements.txt` содержит только production-зависимости.

## Goals / Non-Goals

**Goals:**
- Добавить измерение покрытия кода тестами через `pytest-cov`
- Формализовать dev-зависимости в отдельном файле
- Настроить конфигурацию pytest и coverage в `pyproject.toml`

**Non-Goals:**
- Настройка CI/CD пайплайна (GitHub Actions и т.п.)
- Добавление тестового этапа в Dockerfile
- Написание новых тестов для увеличения покрытия
- Настройка минимального порога покрытия (enforcement)

## Decisions

### 1. `requirements-dev.txt` вместо extras в pyproject.toml

Dev-зависимости выносятся в отдельный `requirements-dev.txt` с `-r requirements.txt` для наследования production-зависимостей.

**Почему:** проект не использует `pyproject.toml` как систему сборки (нет `[build-system]`), только как конфигурацию инструментов. Отдельный файл — минимальное изменение, совместимое с текущим `pip install -r`.

### 2. Конфигурация в `pyproject.toml`

Вся конфигурация pytest и coverage размещается в `pyproject.toml` (`[tool.pytest.ini_options]`, `[tool.coverage.run]`, `[tool.coverage.report]`).

**Почему:** единый файл конфигурации вместо отдельных `pytest.ini` + `.coveragerc`. Стандартный подход для Python-проектов.

### 3. Измеряемые пакеты: `app` и `mini_app`

Coverage настраивается на `app/` и `mini_app/` — это весь application-код проекта.

**Почему:** тесты, миграции (`alembic/`) и конфигурационные файлы не являются целью покрытия.

### 4. asyncio_mode = "auto"

Pytest-asyncio настраивается в режиме `auto` — все async-тесты автоматически распознаются без явного маркера `@pytest.mark.asyncio`.

**Почему:** упрощает написание тестов, убирает необходимость ручной маркировки каждого async-теста.

## Risks / Trade-offs

- **[Совместимость asyncio_mode=auto]** → Проверить, что существующие тесты с явным `@pytest.mark.asyncio` продолжают работать. Режим `auto` обратно совместим — явные маркеры просто игнорируются.
- **[Два файла requirements]** → Разработчики должны знать про `requirements-dev.txt`. Документируется в CLAUDE.md.
