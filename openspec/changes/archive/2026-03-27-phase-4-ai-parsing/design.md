## Context

Текущая реализация (Фаза 3) требует ввода событий в жёстком формате `название, ДД.ММ.ГГГГ, ЧЧ:ММ` через функцию `parse_event_text()` в `services/event_service.py`. Хендлер `bot/handlers/event.py` и callback `bot/callbacks/event_confirm.py` вызывают эту функцию и при `ValueError` показывают подсказку формата.

OpenRouter API уже сконфигурирован в `config.py` (`OPENROUTER_API_KEY`, `OPENROUTER_MODEL`), пакет `openai` в зависимостях — инфраструктура готова.

## Goals / Non-Goals

**Goals:**
- Заменить заглушку парсинга на LLM-извлечение через OpenRouter API
- Поддержать естественный язык: относительные даты («завтра», «в понедельник», «через 3 дня»), разговорные выражения времени («в 8 утра», «после обеда»)
- Обеспечить надёжную работу с retry, таймаутами и валидацией
- Сохранить существующий флоу: парсинг → подтверждение → публикация

**Non-Goals:**
- Тюнинг промптов (Фаза 8)
- Голосовые сообщения (Фаза 5)
- Напоминания (Фаза 6)
- Мультиязычность за пределами русского и английского
- Кэширование ответов LLM

## Decisions

### 1. OpenAI SDK с кастомным base_url

OpenRouter предоставляет OpenAI-совместимый API. Используем `openai.AsyncOpenAI(base_url="https://openrouter.ai/api/v1", api_key=...)` — это даёт типизированный клиент, автоматический retry и streaming без написания HTTP-клиента вручную.

**Альтернатива**: raw `aiohttp` — больше контроля, но требует ручной сериализации, retry, обработки ошибок. Неоправданная сложность.

### 2. Промпты в отдельном модуле `app/prompts/event_parser.py`

Системный и пользовательский промпты выносятся в отдельный модуль с функцией `build_messages(user_text, current_date, weekday)`. Это позволяет итерировать промпты (Фаза 8) без изменения сервисного слоя.

**Альтернатива**: промпты inline в `llm_parser.py` — проще, но сложнее тюнить отдельно.

### 3. Pydantic-модель для валидации ответа LLM

`ParsedEvent(title: str, date: date, time: Optional[time])` — та же структура данных, что и текущий `dataclass`, но через Pydantic для строгой валидации JSON от LLM. Существующий `dataclass ParsedEvent` в `event_service.py` заменяется на Pydantic-модель в `llm_parser.py`. Хендлеры продолжают работать с теми же полями `.title`, `.date`, `.time`.

**Альтернатива**: ручной парсинг JSON + валидация — больше кода, легче ошибиться.

### 4. Retry-стратегия: 3 попытки с экспоненциальным backoff

При HTTP 429 и 5xx — retry с задержками 1s, 2s, 4s. При невалидном JSON от LLM — одна повторная попытка с усиленной инструкцией («Ответь ТОЛЬКО валидным JSON»). Используем `tenacity` или ручной цикл с `asyncio.sleep`.

**Альтернатива**: встроенный retry OpenAI SDK — менее гибкий, не покрывает невалидный JSON.

### 5. Таймаут 15 секунд на запрос

Достаточно для большинства моделей OpenRouter. При превышении — `asyncio.TimeoutError`, пользователь получает «Ошибка соединения, попробуй позже».

### 6. Параметры LLM: temperature 0.1, response_format json_object

Низкая temperature минимизирует «творческую» интерпретацию дат. `response_format: {"type": "json_object"}` гарантирует JSON-вывод на уровне API.

### 7. Изменение хендлеров: замена `parse_event_text` на `parse_event`

В `event.py` и `event_confirm.py` вызов `parse_event_text(text)` заменяется на `await llm_parser.parse_event(text)`. Вместо `ValueError` — кастомное исключение `ParseError` с типом ошибки (invalid_json, timeout, network, rate_limit). Хендлер отображает пользователю соответствующее сообщение.

Сообщение при нажатии «Изменить» меняется с формат-подсказки на «Отправь исправленный вариант» (без формата — пользователь пишет естественным языком).

### 8. `FORMAT_HINT` удаляется

Константа `FORMAT_HINT` в `formatting.py` больше не нужна — пользователь пишет в свободной форме. Все ссылки на неё удаляются из хендлеров.

## Risks / Trade-offs

**[Задержка ответа LLM 1–5 секунд]** → Пользователь будет ждать после отправки сообщения. Митигация: отправка индикатора «typing» через `bot.send_chat_action(chat_id, "typing")` перед вызовом LLM.

**[Невалидный JSON от LLM]** → Одна повторная попытка с усиленной инструкцией. При повторной неудаче — сообщение «Не удалось распознать событие. Попробуй переформулировать.»

**[Rate limit OpenRouter]** → 3 retry с backoff. При исчерпании — «Сервис временно недоступен, попробуй через минуту.»

**[Некорректная интерпретация даты LLM]** → Минимизируется низкой temperature и явным указанием текущей даты/дня недели в промпте. Пользователь видит карточку подтверждения перед публикацией — может отменить или исправить.

**[Стоимость API-вызовов]** → Каждое текстовое сообщение = один LLM-запрос. Для личного бота (~5–20 событий/день) это приемлемо.
