## ADDED Requirements

### Requirement: Команда /reminders показывает текущие настройки

Обработчик команды `/reminders` в `bot/handlers/reminders.py` ДОЛЖЕН отправлять сообщение с текущими настройками напоминаний пользователя и inline-клавиатурой с toggle-кнопками.

#### Scenario: Пользователь вызывает /reminders
- **WHEN** пользователь отправляет команду `/reminders`
- **THEN** бот отвечает сообщением «⏰ Настройки напоминаний:\n\nВыбери, за сколько до события напоминать:» с inline-клавиатурой, где каждая кнопка отображает ☑ или ☐ в зависимости от текущей настройки

#### Scenario: Отображение настроек по умолчанию
- **WHEN** пользователь с дефолтными настройками вызывает `/reminders`
- **THEN** клавиатура отображает: `☑ За 1 день`, `☑ За 2 часа`, `☐ За 1 час`, `☐ За 30 минут`, `☑ За 15 минут`, `☑ В момент события`

### Requirement: Inline-клавиатура с toggle-кнопками

Клавиатура `keyboards/reminder_settings.py` ДОЛЖНА содержать 6 кнопок (по одной на каждый интервал), каждая с callback data формата `reminder_toggle:{key}`.

#### Scenario: Формат кнопок
- **WHEN** генерируется клавиатура для пользователя с `reminder_settings = {"1d": true, "2h": false, ...}`
- **THEN** кнопка для `1d` отображается как `☑ За 1 день` с callback data `reminder_toggle:1d`, кнопка для `2h` — как `☐ За 2 часа` с callback data `reminder_toggle:2h`

#### Scenario: Расположение кнопок
- **WHEN** генерируется клавиатура напоминаний
- **THEN** 6 кнопок размещаются по одной в каждом ряду, в порядке: 1d, 2h, 1h, 30m, 15m, 0m

### Requirement: Переключение настройки по нажатию кнопки

Callback-обработчик `callbacks/reminder_toggle.py` ДОЛЖЕН инвертировать выбранную настройку в `users.reminder_settings` и обновить клавиатуру.

#### Scenario: Включение выключенного интервала
- **WHEN** пользователь с `reminder_settings = {"1h": false, ...}` нажимает кнопку `reminder_toggle:1h`
- **THEN** `reminder_settings["1h"]` устанавливается в `true`, клавиатура обновляется через `edit_message_reply_markup`, кнопка `1h` отображается как `☑ За 1 час`

#### Scenario: Выключение включённого интервала
- **WHEN** пользователь с `reminder_settings = {"1d": true, ...}` нажимает кнопку `reminder_toggle:1d`
- **THEN** `reminder_settings["1d"]` устанавливается в `false`, клавиатура обновляется, кнопка `1d` отображается как `☐ За 1 день`

#### Scenario: Невалидный ключ в callback
- **WHEN** callback data содержит `reminder_toggle:invalid_key`
- **THEN** callback отвечает `answer()` без изменений, ошибка логируется

### Requirement: Чтение настроек с forward-compatibility

При чтении `reminder_settings` из БД код ДОЛЖЕН использовать `.get(key, False)` для каждого ключа, чтобы отсутствующие ключи трактовались как `False`.

#### Scenario: Пользователь с неполным JSONB
- **WHEN** у пользователя в `reminder_settings` отсутствует ключ `0m`
- **THEN** `.get("0m", False)` возвращает `False`, интерфейс отображает `☐ В момент события`

### Requirement: Метки интервалов на русском языке

Каждый интервал ДОЛЖЕН иметь фиксированную человекочитаемую метку для отображения в клавиатуре и сообщениях.

#### Scenario: Маппинг ключей к меткам
- **WHEN** отображается интерфейс настроек
- **THEN** используются метки: `1d` → «За 1 день», `2h` → «За 2 часа», `1h` → «За 1 час», `30m` → «За 30 минут», `15m` → «За 15 минут», `0m` → «В момент события»
