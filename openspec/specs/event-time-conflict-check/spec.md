# Capability: event-time-conflict-check

## Purpose

Проверка конфликтов по времени при создании событий: поиск существующих событий в пространстве, пересекающихся по времени с создаваемым, и форматирование предупреждения для пользователя.

## Requirements

### Requirement: Поиск конфликтующих событий по времени

Сервисная функция `find_conflicting_events` в `event_service.py` ДОЛЖНА находить события в указанном пространстве, которые пересекаются по времени с создаваемым событием. Окно конфликта — +-2 часа от времени нового события.

Функция ДОЛЖНА принимать: `session`, `space_id`, `event_date`, `event_time`, и опциональный `exclude_event_id` (для исключения редактируемого события).

Функция ДОЛЖНА возвращать список объектов `Event`, отсортированных по `event_time`.

#### Scenario: Конфликтующее событие найдено в окне 2 часа
- **WHEN** в пространстве есть событие «Ужин» на 19:00, и создаётся событие на ту же дату на 18:00
- **THEN** функция возвращает список с событием «Ужин» (разница 1 час < 2 часа)

#### Scenario: Событие за пределами окна 2 часа
- **WHEN** в пространстве есть событие «Ужин» на 19:00, и создаётся событие на ту же дату на 10:00
- **THEN** функция возвращает пустой список (разница 9 часов > 2 часа)

#### Scenario: Событие ровно на границе окна (2 часа)
- **WHEN** в пространстве есть событие на 19:00, и создаётся событие на ту же дату на 17:00
- **THEN** функция возвращает пустой список (разница ровно 2 часа — граница не включается)

#### Scenario: Новое событие без времени — проверка не выполняется
- **WHEN** создаётся событие без указания времени (`event_time IS NULL`)
- **THEN** функция возвращает пустой список (проверка пропускается)

#### Scenario: Существующее событие без времени — не считается конфликтом
- **WHEN** в пространстве есть событие без времени на ту же дату, и создаётся событие на 15:00
- **THEN** функция возвращает пустой список (событие без времени не конфликтует)

#### Scenario: Несколько конфликтующих событий
- **WHEN** в пространстве есть события «Обед» на 13:00 и «Звонок» на 14:30, и создаётся событие на ту же дату на 14:00
- **THEN** функция возвращает оба события, отсортированные по времени: «Обед» (13:00), «Звонок» (14:30)

#### Scenario: Конфликт на другой дате — не найден
- **WHEN** в пространстве есть событие на 15:00 на завтра, и создаётся событие на 15:00 на послезавтра
- **THEN** функция возвращает пустой список (разные даты)

#### Scenario: Исключение редактируемого события
- **WHEN** редактируется событие с `id=X`, в пространстве есть другое событие на 15:00, и новое время — 14:30
- **THEN** функция с `exclude_event_id=X` возвращает список только с другим событием, не включая редактируемое

#### Scenario: Использование существующего индекса
- **WHEN** выполняется запрос к БД для поиска конфликтов
- **THEN** запрос ДОЛЖЕН использовать условие `space_id = :space_id AND event_date = :date` для задействования индекса `idx_events_space_date`

### Requirement: Форматирование предупреждения о конфликтах

Функция `format_conflict_warning` в `formatting.py` ДОЛЖНА форматировать предупреждение о конфликтующих событиях для отображения в Telegram-сообщении.

#### Scenario: Одно конфликтующее событие с временем
- **WHEN** найдено одно конфликтующее событие «Ужин» на 19:00
- **THEN** возвращается текст: `⚠️ На близкое время уже есть события:\n• Ужин (19:00)`

#### Scenario: Несколько конфликтующих событий
- **WHEN** найдены два конфликтующих события: «Обед» (13:00) и «Звонок» (14:30)
- **THEN** возвращается текст: `⚠️ На близкое время уже есть события:\n• Обед (13:00)\n• Звонок (14:30)`
