# Capability: reminder-scheduler

## Purpose

Фоновая задача на основе APScheduler, которая периодически проверяет наступившие напоминания в таблице `scheduled_reminders` и отправляет пользователям Telegram-сообщения с информацией о предстоящих событиях.

## Requirements

### Requirement: Запуск APScheduler вместе с ботом

APScheduler `AsyncIOScheduler` ДОЛЖЕН запускаться в `main.py` до начала polling. Планировщик ДОЛЖЕН работать в том же asyncio event loop, что и бот.

#### Scenario: Успешный запуск планировщика
- **WHEN** бот запускается через `main.py`
- **THEN** APScheduler запускается, регистрируется интервальная задача `process_due_reminders`, затем начинается polling

#### Scenario: Интервал из конфигурации
- **WHEN** переменная `REMINDER_CHECK_INTERVAL_SECONDS` равна 30
- **THEN** задача `process_due_reminders` выполняется каждые 30 секунд

### Requirement: Запрос наступивших напоминаний

Задача `process_due_reminders` в `scheduler/jobs.py` ДОЛЖНА на каждом тике запрашивать неотправленные напоминания с наступившим временем.

#### Scenario: Запрос наступивших напоминаний
- **WHEN** выполняется тик планировщика
- **THEN** выполняется запрос `SELECT * FROM scheduled_reminders WHERE sent = FALSE AND remind_at <= NOW() ORDER BY remind_at LIMIT 50`

#### Scenario: Нет наступивших напоминаний
- **WHEN** выполняется тик планировщика и нет напоминаний с `sent = FALSE AND remind_at <= NOW()`
- **THEN** задача завершается без отправки сообщений

### Requirement: Отправка Telegram-сообщений напоминаний

Для каждого наступившего напоминания планировщик ДОЛЖЕН загрузить связанное событие и отправить форматированное Telegram-сообщение пользователю.

Сообщение напоминания ДОЛЖНО включать дату события для контекста. Для напоминаний с временем (не `1d`) — показывать и относительный интервал, и абсолютное время. Для напоминания `1d` — показывать дату события.

#### Scenario: Напоминание за 15 минут — событие с временем
- **WHEN** найдено наступившее напоминание типа `15m` для события «Ужин с родителями» в 19:00 в пространстве «Семья»
- **THEN** пользователю отправляется сообщение: `🔔 Напоминание!\n\n📝 Ужин с родителями\n⏰ Через 15 минут (19:00)\n📅 28 марта, суббота\n📍 Пространство: Семья`

#### Scenario: Напоминание за 1 день — событие с временем
- **WHEN** найдено наступившее напоминание типа `1d` для события «Ужин с родителями» завтра в 19:00
- **THEN** пользователю отправляется сообщение: `🔔 Напоминание!\n\n📝 Ужин с родителями\n⏰ Завтра в 19:00\n📅 28 марта, суббота\n📍 Пространство: Семья`

#### Scenario: Напоминание за 1 день — событие без времени
- **WHEN** найдено наступившее напоминание типа `1d` для события без времени «День рождения Ани»
- **THEN** пользователю отправляется сообщение: `🔔 Напоминание!\n\n📝 День рождения Ани\n📅 Завтра, 5 апреля, воскресенье\n📍 Пространство: Семья`

#### Scenario: Напоминание «Сейчас»
- **WHEN** найдено наступившее напоминание типа `0m` для события «Встреча» в 14:00
- **THEN** пользователю отправляется сообщение: `🔔 Напоминание!\n\n📝 Встреча\n⏰ Сейчас (14:00)\n📍 Пространство: Семья`

#### Scenario: Метка relative_label зависит от reminder_type
- **WHEN** reminder_type = `1d`
- **THEN** relative_label = «Завтра»
- **WHEN** reminder_type = `2h`
- **THEN** relative_label = «Через 2 часа»
- **WHEN** reminder_type = `1h`
- **THEN** relative_label = «Через 1 час»
- **WHEN** reminder_type = `30m`
- **THEN** relative_label = «Через 30 минут»
- **WHEN** reminder_type = `15m`
- **THEN** relative_label = «Через 15 минут»
- **WHEN** reminder_type = `0m`
- **THEN** relative_label = «Сейчас»

### Requirement: Пометка напоминания как отправленного

После попытки отправки (успешной или нет) напоминание ДОЛЖНО быть помечено `sent = TRUE`.

#### Scenario: Успешная отправка
- **WHEN** Telegram-сообщение напоминания успешно доставлено
- **THEN** `scheduled_reminders.sent` устанавливается в `TRUE`

#### Scenario: Ошибка отправки (пользователь заблокировал бота)
- **WHEN** отправка Telegram-сообщения вызывает исключение
- **THEN** `scheduled_reminders.sent` устанавливается в `TRUE`, ошибка логируется с `user_id` и типом ошибки

#### Scenario: Ошибка одного напоминания не блокирует остальные
- **WHEN** в пакете 5 наступивших напоминаний и отправка 3-го вызывает исключение
- **THEN** напоминания 1, 2 отправлены и помечены; 3 помечено с логом ошибки; 4, 5 отправлены и помечены

### Requirement: Собственная DB-сессия для задачи планировщика

Задача `process_due_reminders` ДОЛЖНА создавать собственную async-сессию через `async_session()`, а не использовать middleware aiogram.

#### Scenario: Сессия планировщика
- **WHEN** выполняется тик планировщика
- **THEN** создаётся новая DB-сессия, используется для запросов и обновлений, коммитится и закрывается по завершении тика

### Requirement: Загрузка связанных данных для форматирования

Для каждого напоминания планировщик ДОЛЖЕН загрузить название события, дату, время и название пространства для формирования сообщения.

#### Scenario: Загрузка данных события и пространства
- **WHEN** обрабатывается напоминание с `event_id` указывающим на событие в пространстве «Семья»
- **THEN** из БД загружаются `event.title`, `event.event_date`, `event.event_time`, `space.name` для формирования текста напоминания
