# Capability: events-list

## Purpose

Просмотр списка ближайших событий пространства через команду `/events`: фильтрация будущих событий, сортировка, относительные даты, отображение с/без времени.

## Requirements

### Requirement: Команда /events — список ближайших событий

Команда `/events` ДОЛЖНА отображать ближайшие 10 будущих событий выбранного пространства, отсортированных по дате и времени.

#### Scenario: Просмотр событий пространства
- **WHEN** пользователь отправляет `/events` и состоит в одном пространстве, в котором есть будущие события
- **THEN** бот отвечает нумерованным списком до 10 ближайших событий формата: `📅 Ближайшие события в «{space_name}»:\n\n1. {title} — {date}, {time}`

#### Scenario: Выбор пространства при нескольких
- **WHEN** пользователь отправляет `/events` и состоит в нескольких пространствах
- **THEN** бот показывает inline-клавиатуру выбора пространства с action `"events"`

#### Scenario: Нет будущих событий
- **WHEN** пользователь отправляет `/events` и в пространстве нет будущих событий
- **THEN** бот отвечает: «Нет предстоящих событий»

#### Scenario: Пользователь не в пространстве
- **WHEN** пользователь отправляет `/events` и не состоит ни в одном пространстве
- **THEN** бот отвечает: «Сначала создай пространство через /newspace»

### Requirement: Относительные даты для ближайших событий

События ДОЛЖНЫ отображаться с расширенными относительными датами для лучшей ориентации во времени.

#### Scenario: Событие сегодня
- **WHEN** событие приходится на текущую дату
- **THEN** отображается «сегодня»

#### Scenario: Событие завтра
- **WHEN** событие приходится на следующий день
- **THEN** отображается «завтра»

#### Scenario: Событие послезавтра
- **WHEN** событие приходится на дату через 2 дня
- **THEN** отображается «послезавтра»

#### Scenario: Событие в ближайшие 3-6 дней
- **WHEN** событие приходится на дату через 3-6 дней от текущей
- **THEN** отображается день недели в винительном падеже: «в понедельник», «во вторник», «в среду», «в четверг», «в пятницу», «в субботу», «в воскресенье»

#### Scenario: Событие через неделю и далее
- **WHEN** событие приходится на дату через 7 и более дней
- **THEN** отображается числовая дата формата «5 апреля»

### Requirement: Отображение событий без времени

События без компонента времени ДОЛЖНЫ отображаться без строки ⏰.

#### Scenario: Событие с временем
- **WHEN** у события заполнено поле `event_time`
- **THEN** в строке отображается `{title} — {date}, {time}`

#### Scenario: Событие без времени
- **WHEN** у события `event_time` IS NULL
- **THEN** в строке отображается `{title} — {date}` (без запятой и времени)

### Requirement: Сортировка событий

Список событий ДОЛЖЕН быть отсортирован по `event_date` ASC, затем по `event_time` ASC (события без времени — в начале дня).

#### Scenario: Порядок сортировки
- **WHEN** в пространстве есть событие 5 апреля без времени, событие 5 апреля в 14:00, событие 3 апреля в 10:00
- **THEN** порядок отображения: 3 апреля 10:00, 5 апреля (без времени), 5 апреля 14:00

### Requirement: Только будущие события

Команда `/events` ДОЛЖНА показывать только события с `event_date >= текущая дата`.

#### Scenario: Прошедшие события скрыты
- **WHEN** в пространстве есть события в прошлом и в будущем
- **THEN** отображаются только будущие события (включая сегодняшние)
