# Capability: mini-app-frontend

## Purpose

Telegram Mini App фронтенд: React SPA с кастомной дизайн-системой (без @telegram-apps/telegram-ui), страницы для пространств, событий, настроек напоминаний, навигация и обработка состояний.

## Requirements

### Requirement: Инициализация Mini App

Приложение SHALL инициализировать Telegram WebApp (`ready()`, `expand()`), определять тему (`colorScheme`) и применять соответствующий набор CSS-переменных дизайн-системы. Приложение SHALL НЕ использовать `@telegram-apps/telegram-ui` и `AppRoot`. Обёртка SHALL применять системный шрифт (`-apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif`).

#### Scenario: Запуск в light mode
- **WHEN** Mini App открывается с `colorScheme: 'light'`
- **THEN** применяется светлая тема с bg #f2f2f7, text #1a1a1a

#### Scenario: Запуск в dark mode
- **WHEN** Mini App открывается с `colorScheme: 'dark'`
- **THEN** применяется тёмная тема с инвертированными значениями, контраст текста >= 4.5:1

### Requirement: Экран списка пространств

Экран `/` SHALL отображать список пространств пользователя в Section без заголовка. Каждое пространство SHALL отображаться как ListItem с Avatar (имя пространства, id), названием, подписью "N участн. * M событий", ChevronRight справа. Под списком пространств SHALL отображаться Section "Настройки" с ListItem "Напоминания" (IconBell, подпись "Настроить интервалы"). При загрузке SHALL показываться loading-состояние, при ошибке -- error с кнопкой повтора, при пустом списке -- empty-состояние. В хедере справа SHALL отображаться кнопка "+" (IconPlus) для создания нового пространства.

#### Scenario: Список из 3 пространств
- **WHEN** у пользователя 3 пространства
- **THEN** отображаются 3 ListItem с аватарами, разделённые Divider, секция "Настройки" ниже, и кнопка "+" в хедере

#### Scenario: Переход в пространство
- **WHEN** пользователь нажимает на пространство "Семья"
- **THEN** навигация на `/spaces/1`

#### Scenario: Переход к созданию пространства
- **WHEN** пользователь нажимает "+" в хедере
- **THEN** навигация на `/spaces/new`

#### Scenario: Переход к напоминаниям
- **WHEN** пользователь нажимает "Напоминания"
- **THEN** навигация на `/settings/reminders`

### Requirement: Страница пространства с событиями

Mini App ДОЛЖЕН отображать список будущих событий выбранного пространства с возможностью управления.

#### Scenario: Просмотр событий пространства
- **WHEN** пользователь открывает страницу пространства
- **THEN** отображается список будущих событий с названием, датой, временем и именем создателя

#### Scenario: Отображение относительных дат
- **WHEN** событие приходится на ближайшие 7 дней
- **THEN** дата отображается относительно: «сегодня», «завтра», «послезавтра», день недели

#### Scenario: Отличие своих и чужих событий
- **WHEN** пользователь просматривает список событий
- **THEN** для своих событий отображается кнопка/свайп для редактирования и удаления

#### Scenario: Пустой список событий
- **WHEN** в пространстве нет будущих событий
- **THEN** отображается сообщение «Нет предстоящих событий. Отправь текст или голосовое сообщение в бот, чтобы создать событие.»

### Requirement: Редактирование события в Mini App

Владелец события ДОЛЖЕН иметь возможность редактировать название, дату и время через удобную форму.

#### Scenario: Открытие формы редактирования
- **WHEN** владелец нажимает кнопку редактирования на своём событии
- **THEN** открывается форма с текущими значениями: название (текстовое поле), дата (date picker), время (time picker или чекбокс «весь день»)

#### Scenario: Сохранение изменений
- **WHEN** владелец изменяет поля и нажимает «Сохранить»
- **THEN** изменения отправляются на `PUT /api/events/:id`, список обновляется, показывается подтверждение

#### Scenario: Валидация формы
- **WHEN** владелец пытается сохранить пустое название или название > 500 символов
- **THEN** отображается ошибка валидации, запрос не отправляется

#### Scenario: Ошибка сервера при сохранении
- **WHEN** API возвращает ошибку при сохранении
- **THEN** отображается сообщение об ошибке, форма остаётся открытой

### Requirement: Удаление события в Mini App

Владелец события ДОЛЖЕН иметь возможность удалить своё событие с подтверждением.

#### Scenario: Запрос подтверждения удаления
- **WHEN** владелец нажимает кнопку удаления на своём событии
- **THEN** отображается диалог подтверждения: «Удалить событие «{title}»? Это действие нельзя отменить.»

#### Scenario: Подтверждение удаления
- **WHEN** владелец подтверждает удаление
- **THEN** событие удаляется через `DELETE /api/events/:id`, список обновляется

#### Scenario: Отмена удаления
- **WHEN** владелец отменяет удаление
- **THEN** диалог закрывается, событие остаётся

### Requirement: Страница управления пространством

Mini App ДОЛЖЕН отображать страницу с информацией о пространстве и списком участников.

#### Scenario: Просмотр участников пространства
- **WHEN** пользователь переходит на страницу управления пространством
- **THEN** отображаются: название пространства, ссылка для приглашения, список участников (имя, username, роль)

#### Scenario: Удаление участника администратором
- **WHEN** администратор нажимает «Удалить» рядом с участником
- **THEN** отображается подтверждение, после подтверждения участник удаляется через API

#### Scenario: Удаление участника не-администратором
- **WHEN** пользователь с ролью member просматривает список участников
- **THEN** кнопки удаления участников НЕ отображаются

#### Scenario: Удаление пространства администратором
- **WHEN** администратор нажимает «Удалить пространство»
- **THEN** отображается диалог подтверждения с предупреждением о каскадном удалении, после подтверждения пространство удаляется

#### Scenario: Копирование invite-ссылки
- **WHEN** пользователь нажимает на invite-ссылку
- **THEN** ссылка копируется в буфер обмена, показывается подтверждение

### Requirement: Экран напоминаний

Экран `/settings/reminders` SHALL отображать Section "Интервалы напоминаний" с 6 строками: каждая содержит текст интервала и Toggle. Под секцией SHALL быть подпись (13px, text-secondary): "Настройки применяются ко всем новым событиям во всех пространствах. Уже созданные напоминания не изменятся." Toggle SHALL вызывать API `PUT /api/user/reminders` при переключении. При ошибке SHALL откатывать состояние toggle.

#### Scenario: Переключение напоминания
- **WHEN** пользователь включает toggle "За 1 час"
- **THEN** API вызывается с `{"1h": true}`, toggle отображается включённым

#### Scenario: Ошибка сохранения
- **WHEN** API возвращает ошибку при переключении
- **THEN** toggle возвращается в предыдущее состояние

### Requirement: Экран создания пространства

Экран `spaceCreate` (маршрут `/spaces/new`) SHALL отображаться при нажатии кнопки "+" в хедере экрана списка пространств. Хедер SHALL иметь заголовок «Новое пространство». Экран SHALL содержать:
- Section с заголовком «Название пространства»
- Input с placeholder «Например, "Семья" или "Работа"», autofocus
- Кнопку «Создать» (primary, full-width, borderRadius 12px) — disabled при пустом названии (opacity 0.4)
- Подпись под кнопкой (13px, text-secondary): «Вы станете администратором. Пригласите участников по ссылке после создания.»

При нажатии «Создать» SHALL отправляться `POST /api/spaces` с `{"name": "<trimmed value>"}`. При успехе — навигация на главный экран `/` и toast «Пространство создано». При ошибке — toast с текстом ошибки.

#### Scenario: Создание пространства через Mini App
- **WHEN** пользователь вводит «Семья» и нажимает «Создать»
- **THEN** отправляется `POST /api/spaces {"name": "Семья"}`, при успехе навигация на `/`, toast «Пространство создано»

#### Scenario: Кнопка "+" на экране пространств
- **WHEN** пользователь на экране `/` (список пространств)
- **THEN** в хедере справа отображается кнопка "+" (IconPlus, color #378ADD)

#### Scenario: Нажатие "+"
- **WHEN** пользователь нажимает "+"
- **THEN** навигация на `/spaces/new`, поле ввода пустое, autofocus

#### Scenario: Пустое название — кнопка disabled
- **WHEN** поле ввода пустое или содержит только пробелы
- **THEN** кнопка «Создать» disabled, opacity 0.4

#### Scenario: Ошибка API при создании
- **WHEN** `POST /api/spaces` возвращает ошибку
- **THEN** отображается toast с текстом ошибки, форма остаётся открытой

### Requirement: Навигация и BackButton

Приложение SHALL использовать React Router с маршрутами: `/`, `/spaces/new`, `/spaces/:id`, `/spaces/:id/edit`, `/spaces/:id/members`, `/spaces/:id/events/new`, `/events/:id`, `/settings/reminders`. Telegram BackButton SHALL синхронизироваться с навигацией: скрыт на `/`, показан на всех остальных маршрутах. Нажатие BackButton SHALL вызывать `navigate(-1)`.

#### Scenario: Навигация назад с BackButton
- **WHEN** пользователь на экране `/spaces/:id/events/new` нажимает Telegram BackButton
- **THEN** происходит навигация на `/spaces/:id`

#### Scenario: BackButton скрыт на главном экране
- **WHEN** пользователь на маршруте `/`
- **THEN** Telegram BackButton скрыт

### Requirement: Загрузка данных и состояния

Mini App ДОЛЖЕН показывать состояния загрузки и обрабатывать ошибки API.

#### Scenario: Загрузка данных
- **WHEN** страница запрашивает данные из API
- **THEN** отображается скелетон/спиннер до получения ответа

#### Scenario: Ошибка загрузки
- **WHEN** API-запрос завершается ошибкой
- **THEN** отображается сообщение об ошибке с кнопкой «Повторить»

#### Scenario: Ошибка авторизации
- **WHEN** API возвращает 401
- **THEN** отображается сообщение «Ошибка авторизации. Попробуйте закрыть и открыть приложение заново.»

### Requirement: Удаление зависимости @telegram-apps/telegram-ui

Проект SHALL НЕ импортировать и НЕ использовать пакет `@telegram-apps/telegram-ui`. Все UI-компоненты (Section, Cell, Button, Input, Switch, List, Badge, Spinner, Placeholder) SHALL быть заменены на кастомные компоненты из miniapp-ui-components. Пакет SHALL быть удалён из `package.json`.

#### Scenario: Сборка без @telegram-apps/telegram-ui
- **WHEN** проект собирается (`npm run build`)
- **THEN** сборка проходит успешно без пакета `@telegram-apps/telegram-ui`

### Requirement: Loading и Error состояния без Telegram UI

Система SHALL предоставлять компоненты LoadingView (спиннер + текст "Загрузка..."), ErrorView (текст ошибки + кнопка "Повторить"), EmptyView (текст сообщения) без использования Telegram UI Spinner/Placeholder. Компоненты SHALL использовать кастомные стили из дизайн-системы.

#### Scenario: Состояние загрузки
- **WHEN** данные загружаются
- **THEN** отображается анимированный спиннер и текст "Загрузка..." по центру

#### Scenario: Состояние ошибки с повтором
- **WHEN** загрузка завершилась ошибкой
- **THEN** отображается текст ошибки и кнопка "Повторить"
