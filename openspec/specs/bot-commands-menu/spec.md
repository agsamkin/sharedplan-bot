# bot-commands-menu

## Purpose

Регистрация команд бота в Telegram Bot API для отображения в меню пользователя.

## Requirements

### Requirement: Регистрация команд при старте бота

Система ДОЛЖНА регистрировать список пользовательских команд через Telegram Bot API (`set_my_commands`) при каждом запуске бота. Команды ДОЛЖНЫ отображаться в меню Telegram (кнопка «/» рядом с полем ввода).

#### Scenario: Успешная регистрация команд при старте

- **WHEN** бот запускается
- **THEN** вызывается `bot.set_my_commands()` с полным списком пользовательских команд
- **THEN** команды отображаются в меню Telegram для всех пользователей

### Requirement: Состав списка команд

Список зарегистрированных команд ДОЛЖЕН быть обновлён: убраны команды управления (их функции доступны в Mini App), добавлена команда открытия Mini App.

Описания команд ДОЛЖНЫ быть единообразными по стилю и начинаться с глагола или существительного.

#### Scenario: Обновлённый список команд в меню
- **WHEN** пользователь открывает меню команд бота в Telegram
- **THEN** отображаются команды: `start` — «Начать работу», `help` — «Список команд и помощь», `newspace` — «Создать пространство», `spaces` — «Мои пространства», `events` — «Ближайшие события», `app` — «Открыть приложение»

### Requirement: Команда /app для открытия Mini App

Обработчик `/app` ДОЛЖЕН отправлять inline-кнопку с WebApp URL для открытия Mini App.

#### Scenario: Пользователь отправляет /app
- **WHEN** пользователь отправляет `/app`
- **THEN** бот отвечает сообщением с inline-кнопкой WebApp «📱 Открыть приложение»

### Requirement: Улучшенное приветственное сообщение /start

Обработчик `/start` ДОЛЖЕН отправлять информативное приветственное сообщение, которое кратко объясняет возможности бота и даёт чёткий следующий шаг.

#### Scenario: Новый пользователь запускает бота
- **WHEN** пользователь впервые отправляет `/start`
- **THEN** бот отвечает приветственным сообщением, включающим: краткое описание возможностей (планирование событий, текст/голос, напоминания), инструкцию создать первое пространство через /newspace, упоминание /help для полного списка команд

#### Scenario: Пользователь с существующими пространствами запускает бота
- **WHEN** пользователь, уже состоящий в пространствах, отправляет `/start` (без deep link)
- **THEN** бот отвечает приветственным сообщением, включающим информацию о текущих пространствах пользователя

### Requirement: Улучшенная справка /help

Обработчик `/help` ДОЛЖЕН показывать структурированную справку, сгруппированную по категориям, с кратким пояснением как пользоваться ботом.

#### Scenario: Пользователь запрашивает помощь
- **WHEN** пользователь отправляет `/help`
- **THEN** бот отвечает справкой, включающей: краткую инструкцию по добавлению событий (просто напиши текст или отправь голосовое), список команд, сгруппированных по категориям (пространства, события, настройки)

### Requirement: Централизованное определение списка команд

Список команд ДОЛЖЕН быть определён в одном месте (`app/bot/commands.py`) как переиспользуемая константа. Модуль `main.py` ДОЛЖЕН импортировать список из этого модуля.

#### Scenario: Единый источник правды для команд

- **WHEN** разработчик добавляет новую команду
- **THEN** достаточно добавить её в один файл `app/bot/commands.py` для обновления меню
