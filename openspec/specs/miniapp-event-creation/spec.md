# Capability: miniapp-event-creation

## Purpose

Экран создания события в Mini App: форма с валидацией, предупреждения о прошедшей дате и конфликтах по времени, отправка через API.

## Requirements

### Requirement: Экран создания события в Mini App
Система SHALL отображать экран создания события по маршруту `/spaces/:id/events/new`. Экран SHALL содержать: Header с заголовком «Новое событие», Section «Детали события» с полями ввода, кнопку «Создать событие», подсказку для пользователя.

#### Scenario: Отображение формы создания
- **WHEN** пользователь переходит на экран `/spaces/:id/events/new`
- **THEN** отображается форма с полями: Название (text input, placeholder «Ужин с друзьями», autoFocus), Дата (input type="date", обязательное), Время (input type="time", обязательное, без пометки «необязательно»)

#### Scenario: Подсказка о уведомлениях
- **WHEN** экран создания события отображается для пространства «Семья»
- **THEN** под кнопкой отображается подсказка (13px, text-secondary): «Все участники пространства "Семья" получат уведомление о новом событии.»

### Requirement: Валидация формы создания события
Система SHALL валидировать поля формы перед отправкой. Название MUST быть непустым (после trim) и не длиннее 500 символов. Дата MUST быть заполнена. Время MUST быть заполнено. Кнопка «Создать событие» SHALL быть disabled (opacity 0.4) при невалидной форме.

#### Scenario: Пустое название -- кнопка disabled
- **WHEN** поле названия пустое или содержит только пробелы
- **THEN** кнопка «Создать событие» disabled, opacity 0.4

#### Scenario: Отсутствует дата -- кнопка disabled
- **WHEN** поле даты не заполнено
- **THEN** кнопка «Создать событие» disabled, opacity 0.4

#### Scenario: Отсутствует время -- кнопка disabled
- **WHEN** поле времени не заполнено
- **THEN** кнопка «Создать событие» disabled, opacity 0.4

#### Scenario: Название слишком длинное
- **WHEN** пользователь вводит название длиннее 500 символов
- **THEN** отображается ошибка валидации, кнопка disabled

#### Scenario: Валидная форма
- **WHEN** название непустое (до 500 символов), дата заполнена и время заполнено
- **THEN** кнопка «Создать событие» активна (opacity 1)

### Requirement: Предупреждение о прошедшей дате
Система SHALL отображать предупреждение, если выбранная дата в прошлом. Предупреждение SHALL быть информативным и не блокировать создание.

#### Scenario: Выбрана прошедшая дата
- **WHEN** пользователь выбирает дату раньше текущей
- **THEN** под полем даты отображается предупреждение (оранжевый текст): «Выбранная дата уже прошла»

#### Scenario: Выбрана будущая дата
- **WHEN** пользователь выбирает дату сегодня или позднее
- **THEN** предупреждение не отображается

### Requirement: Предупреждение о конфликтах по времени
Система SHALL проверять конфликты с существующими событиями пространства (+-2 часа на ту же дату) и отображать предупреждение. Предупреждение не блокирует создание.

#### Scenario: Конфликт с существующим событием
- **WHEN** пользователь выбирает дату и время, и в пространстве есть события в окне +-2 часа
- **THEN** отображается предупреждение с перечнем конфликтующих событий: «На близкое время уже есть события: {список}»

#### Scenario: Нет конфликтов
- **WHEN** на выбранные дату и время нет конфликтующих событий
- **THEN** предупреждение не отображается

### Requirement: Отправка формы и создание события
Система SHALL при нажатии «Создать событие» отправлять `POST /api/spaces/:space_id/events` с данными формы. При успехе -- навигация назад и Toast «Событие создано». При ошибке -- Toast с текстом ошибки.

#### Scenario: Успешное создание
- **WHEN** пользователь заполняет форму и нажимает «Создать событие»
- **THEN** отправляется `POST /api/spaces/:space_id/events` с телом `{"title": "<trimmed>", "event_date": "<YYYY-MM-DD>", "event_time": "<HH:MM>"}`, при успехе -- навигация назад, Toast «Событие создано»

#### Scenario: Ошибка API при создании
- **WHEN** `POST /api/spaces/:space_id/events` возвращает ошибку
- **THEN** отображается Toast с текстом ошибки, форма остаётся открытой

#### Scenario: Повторная отправка предотвращается
- **WHEN** пользователь нажимает «Создать событие» во время отправки запроса
- **THEN** кнопка disabled до завершения запроса (предотвращение дублирования)
