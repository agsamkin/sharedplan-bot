# Capability: voice-event-handler

## Purpose

Telegram-обработчик голосовых сообщений: скачивание аудио, вызов STT, интеграция с event creation flow, карточка подтверждения с транскрипцией.

## Requirements

### Requirement: Перехват голосовых сообщений Telegram

Обработчик `bot/handlers/voice.py` ДОЛЖЕН перехватывать все голосовые сообщения (`Message.voice`) и запускать voice-to-event пайплайн.

#### Scenario: Пользователь отправляет голосовое сообщение
- **WHEN** пользователь отправляет голосовое сообщение боту
- **THEN** запускается пайплайн: скачивание аудио → транскрипция → LLM-парсинг → выбор пространства → карточка подтверждения с транскрипцией

#### Scenario: Индикатор обработки
- **WHEN** пользователь отправляет голосовое сообщение
- **THEN** бот отправляет `chat_action: typing` перед началом обработки

### Requirement: Скачивание аудио в память

Обработчик ДОЛЖЕН скачивать аудиофайл из Telegram Bot API в `BytesIO` (в память, не на диск) через `bot.download(file)`.

#### Scenario: Скачивание голосового сообщения
- **WHEN** бот получает голосовое сообщение с `voice.file_id`
- **THEN** аудиофайл скачивается в `BytesIO` и его байты передаются в `speech_to_text.transcribe()`

### Requirement: Интеграция с существующим event flow

После получения транскрипции обработчик ДОЛЖЕН передать текст в `llm_parser.parse_event()` и далее использовать общую логику создания события (выбор пространства, карточка подтверждения, FSM).

#### Scenario: Успешная транскрипция и парсинг
- **WHEN** Nexara вернул текст, LLM успешно извлёк событие
- **THEN** отображается карточка подтверждения с транскрипцией и данными события, FSM переходит в состояние ожидания подтверждения

#### Scenario: Успешная транскрипция, ошибка парсинга LLM
- **WHEN** Nexara вернул текст, но LLM не смог извлечь событие
- **THEN** бот показывает транскрипцию и сообщение об ошибке парсинга (те же сообщения, что и для текстового ввода)

#### Scenario: Транскрипция сохраняется в FSM-данных
- **WHEN** транскрипция получена и парсинг успешен
- **THEN** транскрипция сохраняется в FSM-данных как `transcript` для отображения в карточке подтверждения

#### Scenario: raw_input содержит транскрипцию
- **WHEN** голосовое событие подтверждено и сохранено
- **THEN** поле `events.raw_input` содержит транскрибированный текст

### Requirement: Обработка отключённого STT-сервиса

При получении голосового сообщения, если сервис STT отключён (`service_disabled`), бот ДОЛЖЕН отправить пользователю понятное сообщение о недоступности функции.

#### Scenario: Nexara отключён — голосовое сообщение

- **WHEN** пользователь отправляет голосовое сообщение и `transcribe()` выбрасывает `TranscriptionError` с типом `service_disabled`
- **THEN** бот отвечает: «Функция голосовых сообщений отключена. Напиши текстом или создай событие в приложении.»

### Requirement: Обработка отключённого LLM при голосовом вводе

При получении голосового сообщения, если транскрипция успешна, но LLM-парсинг отключён (`service_disabled`), бот ДОЛЖЕН показать транскрипцию и предложить альтернативу.

#### Scenario: Nexara работает, OpenRouter отключён — голосовое сообщение

- **WHEN** пользователь отправляет голосовое сообщение, `transcribe()` возвращает текст, но `parse_event()` выбрасывает `ParseError` с типом `service_disabled`
- **THEN** бот отвечает с текстом транскрипции и сообщением: «Автоматическое распознавание событий отключено. Создай событие вручную в приложении.»

### Requirement: Обработка ошибок транскрипции

При ошибках Nexara API бот ДОЛЖЕН отправить понятное дружелюбное сообщение и предложить альтернативу.

#### Scenario: Nexara недоступен или таймаут
- **WHEN** `transcribe()` выбрасывает `TranscriptionError` с типом `timeout` или `service_unavailable`
- **THEN** бот отвечает: «Не удалось распознать голосовое сообщение. Попробуй записать ещё раз или напиши текстом.»

#### Scenario: Пустой результат транскрипции
- **WHEN** `transcribe()` выбрасывает `TranscriptionError` с типом `empty_result`
- **THEN** бот отвечает: «Не удалось разобрать речь — возможно, запись слишком тихая. Попробуй записать ещё раз или напиши текстом.»

#### Scenario: Ошибка авторизации Nexara
- **WHEN** `transcribe()` выбрасывает `TranscriptionError` с типом `auth_error`
- **THEN** бот отвечает: «Сервис распознавания речи временно недоступен. Попробуй позже или напиши текстом.», ошибка логируется как ERROR

### Requirement: Регистрация роутера

Роутер `voice.router` ДОЛЖЕН быть зарегистрирован в приложении. Обработчик голосовых сообщений НЕ ДОЛЖЕН конфликтовать с обработчиком текстовых сообщений.

#### Scenario: Текстовое и голосовое сообщения обрабатываются разными хендлерами
- **WHEN** пользователь отправляет текст
- **THEN** срабатывает `event.py`, а не `voice.py`

#### Scenario: Голосовое сообщение обрабатывается voice handler
- **WHEN** пользователь отправляет голосовое сообщение
- **THEN** срабатывает `voice.py`, а не `event.py`
