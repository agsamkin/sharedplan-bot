# Shared Plan Bot — Концепция проекта

## Назначение

Shared Plan Bot — это **Telegram-бот для совместного планирования событий** в небольших группах. Он решает конкретную задачу: координация планов между людьми (семья, пара, друзья, небольшая команда) без необходимости устанавливать новое приложение, разбираться в новом интерфейсе или выходить из Telegram.

Пользователи создают **spaces (пространства)** — общие календари — и добавляют события, отправляя сообщения в свободной форме (текстом или голосом). Бот с помощью AI извлекает структурированные данные о событии (название, дата, время), запрашивает подтверждение, уведомляет всех участников пространства и отправляет персонализированные напоминания.

Система разворачивается как **Docker Compose стек** (бот + PostgreSQL) одной командой `docker compose up`.

---

## Основные сущности

### Space (пространство)

**Space** — это общий календарь, объединяющий участников. У каждого пространства есть название и уникальная invite-ссылка. Пространства — основная организационная единица: все события принадлежат конкретному пространству, все уведомления привязаны к его участникам.

Пространства рассчитаны на небольшие группы: семьи, пары, компании друзей или команды до ~20 человек. Один пользователь может состоять в нескольких пространствах одновременно (например, «Семья» и «Работа»).

### Участники

**Участник** — это пользователь Telegram, присоединившийся к пространству по invite-ссылке. У каждого участника есть роль внутри пространства:

- **Администратор** — пользователь, создавший пространство. Может управлять участниками (удалять) и удалить само пространство.
- **Участник** — пользователь, присоединившийся по ссылке. Может добавлять события и получать уведомления.

Все участники — независимо от роли — могут добавлять события, просматривать расписание и настраивать личные предпочтения напоминаний.

### Событие (Event)

**Событие** — это запись в пространстве о чём-то, что произойдёт в определённую дату и, опционально, в определённое время. События создаются из текстового или голосового ввода, парсятся LLM, подтверждаются пользователем и публикуются в пространстве.

События без указания времени (например, «День рождения Ани 5 апреля») сохраняются только с датой. Это влияет на поведение напоминаний — см. раздел «Напоминания» ниже.

### Напоминание (Reminder)

**Напоминание** — это запланированное уведомление, отправляемое участнику перед (или в момент) наступления события. Напоминания персонализированы — каждый участник сам выбирает, какие интервалы ему нужны. Система заранее создаёт записи о напоминаниях при публикации события и обрабатывает их через фоновый планировщик.

## Как работает создание событий

### Текстовый ввод

Основной сценарий взаимодействия. Пользователь отправляет боту сообщение в свободной форме в личном чате:

1. Пользователь пишет что-то вроде «Ужин с родителями завтра в 19:00» или «Тренировка в понедельник в 8 утра».
2. Если пользователь состоит в нескольких пространствах — бот предлагает выбрать нужное.
3. Бот отправляет текст в **LLM API** (через OpenRouter) со структурированным промптом, включающим текущую дату и день недели. LLM возвращает JSON: `{title, date, time}`.
4. Бот показывает карточку подтверждения:
   ```
   📌 Событие:
   📝 Ужин с родителями
   📅 27 марта 2026
   ⏰ 19:00

   Опубликовать?
   [✅ Да] [✏️ Изменить] [❌ Отмена]
   ```
5. При подтверждении событие сохраняется, все участники пространства получают уведомление, создаются напоминания.

Шаг подтверждения критически важен — он позволяет поймать ошибки AI-распознавания до того, как событие будет опубликовано.

### Голосовой ввод

Удобная надстройка над текстовым флоу. Пользователь отправляет боту голосовое сообщение:

1. Бот скачивает аудиофайл из Telegram (формат OGG/Opus).
2. Аудио отправляется в **Nexara API** для транскрибации (speech-to-text).
3. Распознанный текст показывается пользователю и далее обрабатывается стандартным текстовым флоу (с шага 2).
4. Карточка подтверждения включает транскрипцию, чтобы пользователь мог проверить, что было распознано:
   ```
   🎤 Распознано: «Ужин с родителями завтра в семь вечера»

   📌 Событие:
   📝 Ужин с родителями
   📅 27 марта 2026
   ⏰ 19:00

   Опубликовать?
   ```

### Уведомление участников

После подтверждения и сохранения события:

1. Все участники пространства **кроме создателя** получают уведомление:
   ```
   📅 Новое событие в «Семья»!

   📝 Ужин с родителями
   ⏰ Завтра в 19:00
   👤 Добавил: Иван
   ```
2. Записи напоминаний создаются для **всех** участников пространства (включая создателя), на основе персональных настроек каждого.

Если отправка уведомления конкретному пользователю не удалась (например, он заблокировал бота), ошибка логируется, но не прерывает процесс для остальных.

## Как работают напоминания

### Доступные интервалы

Система поддерживает шесть интервалов напоминаний:

| Ключ | Интервал |
|---|---|
| `1d` | За 1 день |
| `2h` | За 2 часа |
| `1h` | За 1 час |
| `30m` | За 30 минут |
| `15m` | За 15 минут |
| `0m` | В момент события |

### Настройки по умолчанию

При первом взаимодействии настройки напоминаний инициализируются сбалансированным набором:

- ☑ За 1 день
- ☑ За 2 часа
- ☐ За 1 час
- ☐ За 30 минут
- ☑ За 15 минут
- ☐ В момент события

Это обеспечивает раннее предупреждение, окно для подготовки и финальное напоминание — без перегрузки уведомлениями.

### Персонализация

Каждый участник настраивает предпочтения напоминаний глобально (не для каждого события отдельно). Команда `/reminders` показывает интерактивный интерфейс с toggle-кнопками, где каждый интервал переключается между включённым и выключенным состоянием.

Изменения применяются ко всем **будущим** событиям. Для уже существующих событий напоминания создаются на основе настроек пользователя на момент публикации события. Это осознанное упрощение — ретроактивный пересчёт напоминаний выходит за рамки проекта.

### Механика планирования

Напоминания предвычисляются. При публикации события система перебирает настройки каждого участника пространства и создаёт запись `scheduled_reminder` для каждого включённого интервала — с точной меткой `remind_at`, вычисленной из даты/времени события.

Фоновая задача запускается каждые 30 секунд, выбирает неотправленные напоминания с наступившим временем и отправляет их. Каждое напоминание помечается как отправленное после доставки.

**Краевые случаи**:
- Если `remind_at` оказывается в прошлом на момент создания события (например, событие через 10 минут, но у пользователя включено напоминание за 1 день), такое напоминание не создаётся.
- Для событий без времени (`event_time IS NULL`) создаётся только напоминание `1d`, запланированное на 09:00 накануне.

## Как работают пространства

### Создание

Пользователь отправляет `/newspace`, вводит название, и бот создаёт пространство:
- Генерируется уникальный invite-код (8-символьная URL-safe строка).
- Формируется invite-ссылка вида `https://t.me/<bot_username>?start=join_<invite_code>`.
- Создатель назначается администратором.

### Присоединение

Другие пользователи присоединяются по invite-ссылке. Telegram открывает бота с deep link параметром (`/start join_<invite_code>`). Бот парсит код, находит пространство и добавляет пользователя как участника. Если пользователь уже состоит в пространстве — получает соответствующее сообщение. Все текущие участники уведомляются о новом члене.

### Администрирование

Администратор может:
- Посмотреть информацию о пространстве и список участников (`/space_info`).
- Удалить участника (`/kick @username`).
- Удалить пространство целиком (`/delete_space`, с подтверждением).

Удаление пространства каскадно удаляет все его события и запланированные напоминания.

## Просмотр событий

Команда `/events` показывает ближайшие события пространства (или предлагает выбрать пространство, если пользователь состоит в нескольких). События отображаются нумерованным списком, отсортированным по дате, не более 10 записей:

```
📅 Ближайшие события в «Семья»:

1. Ужин с родителями — завтра, 19:00
2. День рождения Ани — 5 апреля
3. Стоматолог — 10 апреля, 14:30
```

Показываются только будущие события. События без времени отображаются без компонента «⏰». Для ближайших событий используются относительные даты («сегодня», «завтра»).

## Команды бота

| Команда | Описание | Доступ |
|---|---|---|
| `/start` | Приветствие / обработка deep link (присоединение) | Все |
| `/help` | Справка по командам | Все |
| `/newspace` | Создать новое пространство | Все |
| `/spaces` | Список пространств пользователя | Все |
| `/space_info` | Информация о пространстве: участники, invite-ссылка | Участники |
| `/events` | Ближайшие события | Участники |
| `/reminders` | Настройки напоминаний (toggle-интерфейс) | Все |
| `/kick` | Удалить участника из пространства | Админ |
| `/delete_space` | Удалить пространство (с подтверждением) | Админ |

## Часовые пояса

Система работает в **едином часовом поясе: `Europe/Moscow`**. Все даты и время — парсинг ввода, хранение, отображение, планирование напоминаний — используют этот часовой пояс. Поддержка индивидуальных часовых поясов отсутствует.

Это осознанное упрощение, подходящее для личного бота, используемого географически близкой группой людей.

## Возможности развития

Следующие функции **не входят в текущую версию**, но архитектура не должна их исключать:

- **Повторяющиеся события** — правило повторения (формат iCal RRULE) в сущности события.
- **RSVP-реакции** — ответы «приду / не приду» от участников пространства.
- **Комментарии к событиям** — обсуждение внутри события.
- **Интеграция с календарями** — экспорт/импорт через Google Calendar API.
- **Поддержка нескольких часовых поясов** — поле timezone у пользователя.
- **Умный подбор времени** — поиск свободных слотов среди участников.
